### 2026-07-23 Thu - Start With One Agent

- Source: OpenAI, A practical guide to building agents, https://openai.com/business/guides-and-resources/a-practical-guide-to-building-ai-agents/
- Tiny point: Start by maximizing one agent with clearly defined tools because it is easier to understand, evaluate, and maintain; introduce multiple agents only when repeated failures show that complex logic or overlapping tools require distinct roles.
- Hook: Why shouldn’t your first version start with ten agents?
- Narration: Why shouldn’t your first version start with ten agents? OpenAI recommends maximizing one agent first. A single agent can grow by adding clearly defined tools, while the workflow stays manageable. Its context, decisions, tool calls, and errors remain in one trace, so evaluation is easier. You can see whether it chose the wrong tool or misunderstood the instruction. For example, if a support agent cannot check an order, add one well-defined order lookup tool before creating another agent. Multi-agent systems add handoffs, state sync, and result merging—new places to fail. This is a preference, not a ban. Split only when repeated failures show that complex logic or overlapping tools exceed one agent’s role. Tiny Rule: start with one agent, add tools, evaluate failures, then split with evidence. Follow Tiny Agent. Tiny Agent helps you get better at using AI.
