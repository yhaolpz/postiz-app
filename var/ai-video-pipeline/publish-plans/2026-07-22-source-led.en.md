- Source: OpenAI, A practical guide to building agents, https://openai.com/business/guides-and-resources/a-practical-guide-to-building-ai-agents/
- Tiny point: Agent tools can be understood as data, action, or orchestration tools; because one product may combine several capabilities, permissions should be split and reviewed at the operation level.
- Hook: Search and send are both called tools, but their risks are not the same.
- Narration: Search and send are both called tools, but their risks are not the same. OpenAI groups agent tools into three types: data, action, and orchestration. Data tools read context, such as a database, a PDF, or the web. Action tools change an outside system by sending a message, updating a record, or creating a ticket. Orchestration tools treat another specialized agent as a tool and delegate a piece of work. Consider email. Reading a message, drafting a reply, and actually sending it should not share one all-powerful permission. A real tool may combine more than one category, so classify each operation, not only the product name. Tiny Rule: ask whether the operation reads, writes, or delegates; then set permissions and approval to match the risk. Follow Tiny Agent. Tiny Agent helps you get better at using AI.
