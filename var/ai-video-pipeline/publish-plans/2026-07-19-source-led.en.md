- Tiny point: In the evaluator-optimizer workflow, one LLM generates a result while another evaluates it against clear criteria and returns feedback for measurable iterative improvement.
- Hook: Why does "make it better" often produce the same answer in different words?
- Narration: Why does telling a model "make it better" often produce the same answer in different words? Anthropic describes an evaluator-optimizer workflow: one model generates a draft, while another checks it against clear criteria and returns specific feedback. The generator revises, then the evaluator checks again. In literary translation, those criteria might cover tone, terminology, and meaning. In complex research, the evaluator can ask whether the evidence is sufficient or another search is needed. The loop works only when feedback can produce a measurable improvement. If the standard is just "I like this more," the cycle may wander without converging. It also needs a stopping condition, or refinement can continue forever. Tiny Rule: write the evaluation criteria and the stop condition before you start the optimization loop. Follow Tiny Agent. Tiny Agent helps you get better at using AI.
