### 2026-07-12 Sun - Maybe Not an Agent

- Source: Anthropic, Building effective agents, published 2024-12-19, https://www.anthropic.com/engineering/building-effective-agents
- Tiny point: Start with the simplest workable solution, then add agentic complexity only when results show it is worth the extra cost.
- Hook: Does your task really need an agent?
- Narration: Does your task really need an agent? Anthropic's engineering guide says the most successful implementations often use simple, composable patterns, not the most complex agent framework. Why? Because agentic systems add model calls, tool use, decisions, latency, and cost. If the path is fixed, one model call with retrieval and a few examples may already be enough. Tiny example: a weekly report that always fills the same five fields probably needs a clean template, not a free-roaming agent. But a supplier security review can be different: exceptions, follow-up questions, and messy documents may require flexible judgment. The test is not whether the architecture sounds advanced. The test is whether extra complexity creates measurable improvement. Tiny Rule: start simple, evaluate, then add one layer only when the simpler design falls short. Follow Tiny Agent. Learn one AI agent idea every day.
