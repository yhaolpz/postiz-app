### 2026-07-20 Mon - 3 Tasks Worth an Agent

- Source: OpenAI, A practical guide to building agents, https://openai.com/business/guides-and-resources/a-practical-guide-to-building-ai-agents/
- Tiny point: Agents are most useful when a workflow needs complex judgment, has difficult-to-maintain rules, or relies heavily on unstructured data; if a deterministic solution is sufficient, do not add an agent merely to make the system seem more AI-powered.
- Hook: What kind of task is worth the cost and uncertainty of an AI agent?
- Narration: What kind of task is worth the cost and uncertainty of an AI agent? OpenAI recommends looking for three signals. First, complex decisions with exceptions and context, like approving a refund. Second, rule systems that have grown so tangled that every update is costly and error-prone, like a vendor security review. Third, work dominated by unstructured data, such as emails, documents, conversations, or a home insurance claim. These are signals, not a mandate. An agent still adds nondeterminism, evaluation work, and operational cost. If the inputs are clear, the rules are stable, and the output is fixed, ordinary software is usually the better tool. Tiny Rule: consider an agent only when judgment is complex, rules are brittle, or data is messy—and validate the benefit before building. Follow Tiny Agent. Tiny Agent helps you get better at using AI.
