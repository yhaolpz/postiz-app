### 2026-07-13 Mon - Workflow vs Agent

- Source: Anthropic, Building effective agents, published 2024-12-19, https://www.anthropic.com/engineering/building-effective-agents
- Tiny point: In a workflow, code defines the execution path in advance; in an agent, the model dynamically directs the process and tool use.
- Hook: Does using an LLM automatically make it an agent?
- Narration: Does using a large language model automatically make a system an agent? No. Anthropic separates agentic systems into workflows and agents. The key difference is who controls the execution path. In a workflow, code defines the steps in advance, so the input moves along a fixed track. In an agent, the model uses current results to choose the next step and tool. An expense approval with fixed fields and a fixed order is usually more predictable as a workflow. Incident investigation is different: each log result may change what to inspect next. Workflows optimize for predictability. Agents optimize for flexibility. Neither is automatically more advanced. Tiny Rule: if you can draw the whole path beforehand, use a workflow. If you cannot, then consider an agent. Follow Tiny Agent. Learn one AI agent idea every day.
