# Tiny Agent 中文长版预览内容

### 2026-07-11 Sat - 规划者 vs 执行者

- Tiny point: 可靠的 Agent 会把规划和执行分开，并在每一步之后根据真实结果调整计划。
- Hook: 还没理解任务，就开始点击？
- Narration: 如果你的 Agent 还没理解任务就开始点击，它缺的可能不是执行能力，而是规划。可靠的 Agent 会把工作拆成两个角色：规划者决定目标、步骤顺序和停止条件；执行者每次只做一步，并把结果和证据回报回来。比如修复线上 bug，规划者先安排读日志、定位服务、起草方案，最后等待人工批准。执行者先打开日志，返回具体错误，再根据新信息进入下一步。当 Agent 能修改文件、发送消息、花钱，或者任务需要多步判断时，这种拆分最有价值；简单查询则不一定需要。记住 Tiny Rule：先规划，只执行一步，回报结果，再调整计划。关注 Tiny Agent，每天学一个真正能用的 Agent 模式。
- On-screen text: 规划 → 一步 → 回报 → 调整
- Subtitle blocks:
  - 还没理解任务 / 就开始点击？

  - 修改文件、发消息、花钱 / 更需要先规划
