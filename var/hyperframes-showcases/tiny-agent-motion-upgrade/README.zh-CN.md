# Tiny Agent HyperFrames Showcase

## 结论

HyperFrames 对当前 Tiny Agent 最有价值的位置是“后期动效层”，不是替代生图或重新画角色。现有图片模型继续负责程序员、Tiny Agent 和视觉比喻；HyperFrames 负责把这些静态素材组织成更有节奏、更容易理解的成片。

## 样片展示的提升

| 提升点 | 当前链路 | HyperFrames 样片 |
| --- | --- | --- |
| 开头 Hook | 固定标题 + 缓慢放大 | 动态标题、关键词划线、红色警示章 |
| 信息聚焦 | 整张图统一运动 | 图片保持完整，标签和强调层独立引导视线 |
| 切镜 | 白底淡入淡出 | 固定蓝色 marker wipe，形成系列记忆点 |
| 字幕 | 固定字幕框 | 保留字幕框，按语义切换并强调关键词 |
| 品牌感 | 依赖角色和标题 | 增加场次编号、进度轨和统一动效语法 |

## 不建议用 HyperFrames 替代的部分

- 不替代图片模型生成程序员和 Tiny Agent。
- 不用 HTML、SVG 或 Canvas 重画主画面。
- 不把视频改成暗色科技 UI、PPT 或卡片仪表盘。
- 不为了动效增加大量小字和复杂组件。

## 推荐接入方式

```text
内容计划
  -> 图片模型生成关键帧
  -> TTS / 字幕时间轴
  -> HyperFrames 模板套入图片、字幕和音频
  -> 自动 check + snapshot
  -> MP4
  -> Postiz 发布
```

适合先用于三个高收益位置：前 3 秒 Hook、语义切镜、Tiny Rule/CTA。确认完播率提升后，再逐步替换现有 FFmpeg 叠层逻辑。
