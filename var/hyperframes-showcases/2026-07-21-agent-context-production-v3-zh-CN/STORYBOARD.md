# AI 智能体上下文工程：多样化动态中文成品 v3

## 叙事节奏

本片使用“开头点题 → 五个方法章节 → 可执行总结”的结构。每个正文章节固定包含：

1. 一段可朗读的章节开场，说明本章解决什么问题。
2. 四到六个短句核心判断，每幕只承载一个信息焦点。
3. 三条编号小结，在同一画面中按 `1 → 1+2 → 1+2+3` 累计呈现。

## 章节开场

- 第 2 章：判断什么信息值得进入当前窗口。
- 第 3 章：只看指令、工具和示例三个高信号入口。
- 第 4 章：把全部预加载改成需要时再打开。
- 第 5 章：比较压缩、结构化笔记和专门 Agent。
- 第 6 章：完成一份六部分高信号上下文包。

章节开场使用蓝色章节承诺区，并让人物、Tiny Agent 与核心物体共同说明本章任务。

## 内容编辑原则

- 删除重复机制、长背景、产品展开和不能改变行动的细节。
- 权威来源、核心机制、必要边界和可执行方法保留。
- 除固定开头价值承诺外，旁白优先使用一到两句短句完成一个判断。
- 最终实际 TTS 必须在 5–8 分钟内。

## 画面与音频

- 1920×1080、30fps、纸灰白板风格。
- 主标题 92px，长标题 78px，字幕 46px、6px 黑色圆角描边。
- 根据语义在人物、Tiny Agent、双道具、巨大重点词和完整生成插画之间切换；每幕只保留一个中心判断。
- 62 个正文画面中 10 个使用当期生成的完整白板插画，占 16.13%。
- 通过 HTML/CSS/GSAP 提供脉冲、弹跳、短移和轻摆 4 类、24 个重点动作节拍。
- 章节小结使用固定蓝色左栏，右侧三点逐条累积显示。
- 封面结束后同帧显示 1920×52 章节进度条。
- 中文旁白固定为 `zh-CN-YunxiaNeural`、`+40%`。
- 封面持续到第一问句真实可听结束后的下一帧。
- 固定中文 CTA 开始时硬切到独立结束页。
