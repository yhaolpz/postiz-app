# Tiny Agent Shorts 原生动效实验

## 结论

三版都能显著减少原有静态关键帧带来的“课件/PPT 感”，但最适合进入 Tiny Agent 日更生产的是 **B 为主体、吸收 A 的文字强调手法**：用 Tiny Agent 和一个核心物体保持连续动作，再在钩子、公式和结论处加入少量动态排版。

如果只能选一个完整版本，选 B。它在动效感、Tiny Agent 角色连续性和知识表达之间最平衡，也最不像把几张信息卡依次翻页。

## 统一基线

- 知识点：把模糊的“帮我做发布”改成一个可审核的明确产物，再补充上下文、硬约束和审核标准。
- 来源项目：`2026-07-19-15-agent-task-brief-longform-en-US`。
- 标题：`Make Agent Tasks Executable`。
- 三版共用同一 `edge-tts` `en-US-AnaNeural` 音轨、VTT、时长和 CTA；成片均为 1080×1920、30 fps、38.848 秒，背景已与长视频统一为 `#ECECEA` 中性纸灰。
- 为只比较动效方向，实验音轨沿用启动时锁定的 `+20%` 基线；正式 Shorts 从下次运行起固定使用 `+30%`，接入所选动效时会重新生成音轨和 VTT，不复用这条实验音轨。
- 每版包含 7 个正文语义阶段和 1 个独立 CTA 阶段。
- 三版样片只在隔离实验目录内制作且没有上传；日更自动化仅按后续长期规则同步了 `+30%`、实时字幕、`#ECECEA` 和固定 CTA，不接入本实验的 A/B/C 动效实现。

## 三版比较

| 方向 | 信息清晰度 | 动效密度 | 品牌一致性 | 复用成本 | 流水线接入难度 | 判断 |
| --- | --- | --- | --- | --- | --- | --- |
| A 动态排版 | 最高，关键词关系一眼可读 | 高，节奏最强 | 中高，角色存在但不是连续主角 | 中，需按知识结构重新编排文字 | 中 | 适合做钩子、公式和结论强化，不建议整条都只用排版 |
| B 连续角色与物体 | 高，核心“任务包”贯穿全片 | 高，动作有因果连续性 | 最高，Tiny Agent 始终参与 | 中，需建立角色与核心物体动作语法 | 中高 | 最适合作为默认生产骨架 |
| C 工作流界面 | 高，步骤和状态最可验证 | 中高，交互明确但更像产品演示 | 中，Tiny Agent 容易退成界面旁观者 | 高，每个主题都要设计字段和状态 | 高 | 适合确实存在流程或界面的题材，不适合作为通用默认模板 |

## 为什么推荐 B + A

B 解决的是画面的连续性：观众始终知道 Tiny Agent 正在处理同一个“任务包”，输入、限制和验收条件都进入同一对象，最后它变成可执行任务。A 解决的是瞬时理解：`ACTIVITY ≠ OUTCOME`、四项公式和最终转化都能在很短时间内被看懂。

组合后可以保留当前品牌角色和纸灰主题，同时避免两种风险：整条只用 A 会重新变成“更会动的课件”；整条只用 C 会把系列带向工具教程或产品 Demo。

## 生产接入估算

以下是基于当前实验的工程估算，不是交付工时承诺：

- A：约 2–3 个开发日。需要把动态排版抽象为钩子、对比、三项输入、公式、结果五类模板，并接入现有标题/字幕/CTA QA。
- B：约 3–5 个开发日。需要增加“核心物体 + 角色 + 道具”的语义动作描述、素材朝向约束、连续位置状态和越界检查。
- C：约 5–8 个开发日。需要定义可复用的界面字段、状态机、光标轨迹、验证反馈，并为不同知识点编写界面映射。
- 推荐混合方案：约 4–6 个开发日。以 B 的连续动作骨架为主，只接入 A 的三类高价值文字动效：钩子、公式、结论。

当前正式 Shorts 流水线仍以静态语义关键帧为主。要投入生产，关键不是再加几种转场，而是新增一层可由计划文件描述、并能被 QA 验证的语义动作模型。
