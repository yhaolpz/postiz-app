# Augmented LLM First - Programmable Motion Sample

## Beat 1 - Augment one model

The engineer and Tiny Agent change pose as retrieval, tools, and memory connect to one model. The visual point is that capability is attached before additional agents are created.

## Beat 2 - Retrieval

Several documents enter the scene. The Agent searches, rejects noise, and receives one highlighted fact.

## Beat 3 - Tools

Search and refund tools move into the Agent's tool port. The final connection returns an explicit action result.

## Beat 4 - Memory

Order, user, and confirmation state are stored in a persistent memory bank. A compact context card returns to the Agent for the next step.
