# Tiny Agent 中文视频平台发布准备手册

## 目的与边界

本手册用于把已经通过双语准出的 Tiny Agent 中文长视频准备到哔哩哔哩、抖音和小红书的最终发布按钮之前。它记录 2026-07-17 实际操作中确认的页面行为、最短流程、完成证据和故障恢复方式。

- 非开发调试类网页操作统一使用电脑操控连接用户已有登录状态的外部 Chrome。
- 自动化完成视频上传、文案、原生话题、封面和普通必填项；最终发布按钮始终由用户点击。
- 不因浏览器锁屏、标签页切换、页面刷新或单个平台失败而重新制作视频；从当前页面和已有产物恢复。
- 登录、扫码、验证码、2FA、CAPTCHA、实名校验或平台明确要求人工确认时，保留页面并停在该步骤。
- 用户一旦手动接管浏览器，自动化立即停止发送点击和按键；后续只有在重新读取当前标签页、页面和弹窗状态后才能恢复，不能沿用接管前的元素索引、坐标或标签页假设。
- 原创声明的普通开关按既定发布要求设置；出现协议勾选、承诺确认或其它需要用户作出法律/身份声明的弹窗时，不代选、不代确认。若弹窗来自误操作，先核对当前平台和弹窗标题，再使用页面可见的关闭/取消控件恢复，禁止基于旧页面状态直接按键切换标签页。

## 发布输入包

进入浏览器前一次性解析并核对以下输入，避免在三个页面之间反复查文件：

| 输入           | 来源与要求                                                                                                                                                                                                         |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 中文视频       | `renders/video.mp4`，已通过长视频 QA                                                                                                                                                                               |
| 通用标题       | `publish-metadata.zh-CN.json`                                                                                                                                                                                      |
| 小红书标题     | `xiaohongshuTitle`，按页面计数口径不超过 20 个字                                                                                                                                                                   |
| 通用简介       | `manual-publish-copy.zh-CN.md`，不得包含章节标题、时间点、来源机构/标题、`来源：`段落或任何 `http://`、`https://` 外站链接；结尾固定为 `关注 Tiny Agent，成为更擅长使用 AI 的人！` |
| B 站简介       | 通用简介删除整行 `#话题` 文本后的版本；不要先粘贴再返工                                                                                                                                                            |
| 话题清单       | 当期 4-6 个真实相关词，固定保留 `TinyAgent`，平台内逐个绑定为原生实体                                                                                                                                              |
| 中文 16:9 封面 | `thumbnails/thumbnail.zh-CN.png`，用于 B 站“个人空间封面”                                                                                                                                                          |
| 中文 4:3 封面  | `thumbnails/thumbnail.zh-CN.4x3.png`，用于 B 站、小红书和抖音横封面                                                                                                                                                |
| 中文 3:4 封面  | `thumbnails/thumbnail.zh-CN.3x4.png`，用于抖音竖封面                                                                                                                                                               |

输入核对必须先于页面操作。标题、简介或封面路径有误时先修正本地产物；不要在三个页面分别临时改出不同版本。

## 外部浏览器最短流程

### 1. 固定页面与上传顺序

外部 Chrome 固定使用三个标签页：B 站、抖音、小红书。先逐页启动同一中文 MP4 的上传，再利用后台上传时间从最先就绪的页面开始填写。页面发生 DOM 更新后必须重新读取状态，不能复用旧的元素索引。

1. B 站：`https://member.bilibili.com/platform/upload/video/frame?page_from=creative_home_top_upload`
2. 抖音：`https://creator.douyin.com/creator-micro/content/upload`
3. 小红书：`https://creator.xiaohongshu.com/publish/publish?from=menu&target=video`

首次操作和每个关键提交后都读取页面状态；最终检查时使用完整页面状态与截图。优先使用可访问性元素索引，只有页面未暴露元素时才使用坐标，并且坐标点击前必须重新截图。

### 2. 文件选择器

macOS 文件选择器中不要沿多列目录逐层点击，也不要依赖文件名文本的单击状态。统一使用：

1. 按 `Command+Shift+G` 打开路径输入框。
2. 填入目标文件的绝对路径并回车。
3. 确认目标文件被选中、`打开`按钮可用后再点击打开。
4. 返回平台编辑器后检查主预览是否已经切换到目标文件。

这套方式同时用于 MP4 和平台封面 PNG。

## 哔哩哔哩

### 页面流程

1. 上传中文 MP4，等待页面显示上传完成。
2. 填写通用标题，创作声明按实际内容选择 `含AI生成内容`，分区选择 `人工智能`。
3. 上传中文 `4:3` 投稿封面并检查标题、人物和道具没有被裁切。
4. 打开“个人空间封面”设置，上传中文 `16:9` 的 `thumbnail.zh-CN.png`，保存后重新读取两个封面槽位，确认投稿封面仍为 4:3、个人空间封面为 16:9；不能用同一槽位的预览代替另一槽位的证据。
5. 填写不含章节、时间戳、来源、外站链接和 `#话题` 行的 B 站简介；末句使用固定关注句。
6. 通过“搜索更多话题”选择一个最核心、真实匹配的参与话题。
7. 将其余关键词逐个加入“标签”，然后删除平台自动加入或与内容无关的标签，重点检查 `记录`、`剪辑`、`生活记录`。
8. 重新读取页面，确认标题、声明、分区、两个封面槽位、简介、参与话题和标签后，停在 `立即投稿` 前。

### 完成证据

| 检查项       | 必须看到的结果                                                    |
| ------------ | ----------------------------------------------------------------- |
| 视频         | 上传完成                                                          |
| 简介         | 无章节、时间戳、来源、外站链接和 `#话题` 行；固定关注句只出现一次 |
| 话题         | 至少一个真实匹配的原生参与话题                                    |
| 标签         | 仅保留当期相关标签；无 `记录`、`剪辑`、`生活记录`                 |
| 投稿封面     | 中文 4:3 正确预览                                                 |
| 个人空间封面 | 中文 16:9 正确预览；不是空缺、视频帧或 4:3 误用                   |
| 停止点       | `立即投稿`可见但未点击                                            |

## 抖音

### 页面流程

1. 上传中文 MP4，填写通用标题和无章节、无时间戳、无来源、无外站链接的简介，保持公开可见；末句使用固定关注句。
2. 通过 `#添加话题` 逐个搜索并选择话题；页面中出现原生话题实体后才算生效。
3. 先打开“横封面 4:3”槽位，进入封面编辑器并上传 `thumbnail.zh-CN.4x3.png`。
4. 如果上传后中央主预览仍是视频帧，点击新上传的图片缩略图，直到主预览显示正确白板封面，再点击完成。
5. 再打开“竖封面 3:4”槽位，上传 `thumbnail.zh-CN.3x4.png` 并完成。竖封面放在最后处理，因为横封面保存过程可能让竖封面回退为视频帧。
6. 回到主页面同时检查横、竖两个缩略图；任一槽位显示视频帧、比例错误或空缺都必须重做对应槽位。
7. 确认原生话题、公开可见和双封面后，停在 `发布` 前。

### 完成证据

| 检查项 | 必须看到的结果                                       |
| ------ | ---------------------------------------------------- |
| 简介   | 无章节、时间戳、来源和外站链接；固定关注句只出现一次 |
| 话题   | 每个词都是通过 `#添加话题` 产生的原生实体            |
| 横封面 | 4:3 槽位显示 `thumbnail.zh-CN.4x3.png` 的标题与插画  |
| 竖封面 | 3:4 槽位显示 `thumbnail.zh-CN.3x4.png` 的上下构图    |
| 可见性 | 公开                                                 |
| 停止点 | `发布`可见但未点击                                   |

## 小红书

### 页面流程

1. 上传中文 MP4，填写独立的 `xiaohongshuTitle`。
2. 填写前先校验标题不超过 20 个字；填写后再读取页面计数器。超限时改写为完整自然标题，不截断半句。
3. 填写无章节、无时间戳、无来源、无外站链接的简介；末句使用固定关注句。
4. 在简介末尾按“输入一个词 → 选择一个完全匹配的候选 → 重新读取一个 `[话题]` 原生实体 → 清除本次普通文本残留”的顺序逐个绑定。禁止一次性粘贴整行 hashtag，也禁止在已经生成原生实体后用全选重写整个编辑器。全部完成后，原生实体数量必须与计划话题数量一致，且只允许保留原生实体和平台分隔空格。重复词、`##`、`#TinyAgent#AI` 一类连写、没有 `[话题]` 标记的普通 hashtag 均视为失败；无法安全定位残留时，恢复无话题的完整简介，再从第一个话题重新绑定。
5. 上传中文 `4:3` 封面。实际页面要求和本次验证均以 4:3 为准，不使用 3:4 封面。
6. 加入合集 `AI Agent 知识`，并选中原创声明。
7. 重新读取标题计数、完整简介、原生话题、封面、合集和原创声明，并截图确认简介中没有重复或普通 hashtag 残留，停在 `发布` 前。

### 完成证据

| 检查项 | 必须看到的结果                                                                      |
| ------ | ----------------------------------------------------------------------------------- |
| 标题   | 页面计数不超过 20                                                                   |
| 简介   | 无章节、时间戳、来源和外站链接；固定关注句只出现一次                                |
| 话题   | 每个词都有平台原生话题视觉或 `[话题]` 可访问性标记；无重复、连写或普通 hashtag 残留 |
| 封面   | 中文 4:3 正确预览并通过页面质量检查                                                 |
| 合集   | `AI Agent 知识` 已选中                                                              |
| 声明   | 原创声明已选中                                                                      |
| 停止点 | `发布`可见但未点击                                                                  |

## 已踩坑与规避方式

| 现象                                  | 根因                                                     | 以后怎么做                                                                                           | 恢复方式                                                                      |
| ------------------------------------- | -------------------------------------------------------- | ---------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------- |
| 页面打开了但没有登录态                | 使用了 Codex 内置浏览器或独立页面                        | 非开发调试操作只用外部 Chrome 的现有登录态                                                           | 回到原外部浏览器，不重复上传产物                                              |
| 点击了错误标签或封面                  | 页面更新后复用了旧元素索引，或根据旧截图直接点坐标       | 每次 DOM 变化后重新读取状态；优先元素索引                                                            | 立即读取当前标签/预览，恢复误删项后继续                                       |
| `#话题` 文本看起来正确但没有生效      | 只粘贴了普通文本，没有选择平台候选                       | 三个平台都必须走原生话题控件并验证实体                                                               | 光标回到对应词末尾，重新选择候选                                              |
| B 站仍有 `记录`、`剪辑`、`生活记录`   | 平台根据视频自动补了默认标签                             | 标签完成后固定执行一次无关标签清理                                                                   | 逐个删除并重新读取标签列表                                                    |
| B 站简介重复出现 `#话题`              | 已有参与话题和标签，简介又保留 hashtag 行                | 操作前生成不含 `#话题` 行的 B 站简介                                                                 | 整体替换为预先准备的 B 站简介                                                 |
| B 站漏设“个人空间封面”                | 只完成了默认投稿 4:3 封面，把单个封面预览误当成全部完成  | 固定把 B 站视为两个封面槽位：投稿 4:3 后再设置个人空间 16:9，并逐槽复核                              | 打开“个人空间封面”补传 `thumbnail.zh-CN.png`，保存后再次确认投稿 4:3 未被覆盖 |
| 中文简介又出现章节时间                | 误用了 YouTube 英文简介结构或旧版手动文案                | 三个中文平台统一使用无章节、无时间戳版本                                                             | 从本地产物删除章节块，再整体替换三个页面                                      |
| 中文简介出现来源、原文标题或外站链接  | 把内部核验字段或旧版来源模板拼进了发布简介               | 来源只留在 `source.md` 和结构化元数据；页面简介统一执行无 `来源：`、无 `http://`、无 `https://` 检查 | 先修正两个中文文案文件，再整体替换三个页面                                    |
| 小红书出现原生话题和普通 `#文字` 重复 | 选择候选后没有清除输入阶段残留，或一次性粘贴整行 hashtag | 每次选中原生话题后立即清残留；全部完成后逐词读取 `[话题]` 标记并截图                                 | 清空话题区，按“输入一个、选择一个、验证一个”重建                              |
| 用户接管后仍继续发送浏览器按键        | 沿用了接管前的标签页和弹窗状态                           | 检测到用户接管或页面状态冲突时立即停止；恢复前重新读取当前平台、URL、页面和弹窗                      | 不再发送快捷键；等待接管结束后从最新页面状态继续                              |
| 在错误平台误开作品声明或协议弹窗      | 旧标签页假设、旧元素索引或坐标已经失效                   | 每次关键操作前核对平台和控件名称；协议、承诺、身份声明弹窗一律留给用户确认                           | 不选择声明项；核对弹窗后用可见关闭/取消控件退出，再重新读取页面               |
| 抖音只设置了 3:4，横封面提示缺失      | 把“默认竖封面”误认为全部封面已完成                       | 横 4:3 和竖 3:4 必须分别上传                                                                         | 进入横封面编辑器补传 4:3                                                      |
| 抖音封面变成视频帧                    | 点击了 AI 推荐帧，或上传后没有选择新图片                 | 只进入目标槽位的上传编辑器，确认中央主预览                                                           | 重新上传精确文件，必要时点击上传缩略图再完成                                  |
| 抖音修好横封面后竖封面又错了          | 横封面保存会改变或重置竖封面状态                         | 固定顺序为横 4:3 后竖 3:4，竖封面最后保存                                                            | 重开竖封面槽位并补传 3:4                                                      |
| Finder 里看到了文件但“打开”不可用     | 文本获得焦点但文件行未真正选中                           | 统一用 `Command+Shift+G` 和绝对路径                                                                  | 重新输入完整路径，确认选中状态后打开                                          |
| 小红书封面比例选错                    | 早期按竖版平台推断使用 3:4                               | 以实际上传页为准，固定使用中文 4:3                                                                   | 重开封面编辑器替换为 4:3                                                      |
| 锁屏、标签页切换后操作失效            | 电脑操控状态和当前前台页面不一致                         | 恢复后先完整读取当前页面，不沿用旧索引                                                               | 保留原标签页，从已完成字段继续，不重新制作或重传                              |

## 最终跨平台检查

三个页面全部完成后再做一次只读复核：

| 平台   | 文案                                               | 话题/标签                              | 封面                     | 特有设置                      | 停止点     |
| ------ | -------------------------------------------------- | -------------------------------------- | ------------------------ | ----------------------------- | ---------- |
| B 站   | 无章节、时间戳、来源、外链、hashtag 行；固定关注句 | 原生参与话题；相关标签；无默认无关标签 | 投稿 4:3 + 个人空间 16:9 | 含 AI 声明、人工智能分区      | 立即投稿前 |
| 抖音   | 无章节、时间戳、来源、外链；固定关注句             | 全部为原生话题实体                     | 横 4:3 + 竖 3:4          | 公开可见                      | 发布前     |
| 小红书 | 无章节、时间戳、来源、外链；固定关注句；标题 ≤20   | 全部为原生话题实体，无普通文本残留     | 4:3                      | `AI Agent 知识`合集、原创声明 | 发布前     |
