# Tiny Agent 横屏长视频生产基线

## 用途

这份文档是 Tiny Agent 横屏讲解视频的长期生产基线。输入为一篇文章或一组同源资料，输出为结构统一、时长自适应、可自动检查的完整讲解视频。

长视频不按原文段落或文字量平均分配篇幅。系统必须先理解全文，再围绕核心主旨、关键方法、重要边界和高价值案例重新组织内容。成片采用总分总结构，让观众在开头获得明确收益预期，在结尾重新记住中间讲过的关键点。

短视频继续遵循 `scripts/ai-video-pipeline/style-guides/agent-sketchbook.md`，两套规则互不覆盖。

## 输出定位

- 平台：英文版为 YouTube 横屏长视频；中文版为哔哩哔哩、抖音与小红书人工发布准备稿。
- 日更任务固定同时输出英文版和中文版。两个版本复用相同内容地图、内容优先级、章节结构和视觉语义，但分别使用自然英文和自然中文脚本。
- 画布：`1920x1080`，`16:9`，`30fps`。
- 编码：H.264 视频、AAC 音频。
- 主力时长：`9-13 分钟`。
- 自动浮动范围：`6-18 分钟`。
- 超过 `18 分钟` 需要有额外的事实、案例、对比或操作演示支撑，不能靠重复解释延长。
- 风格：白板简笔画、角色驱动、低文字密度；角色演绎与标题页、流程、对比、网格和重点聚焦等信息构图交替使用，避免单一左右构图和纯课件翻页。

## 双语封面

每期必须与双语成片同步生成四张封面：英文 `16:9`、中文 `16:9`、中文 `4:3` 和中文 `3:4`。英文版用于 YouTube 发布；中文 `4:3` 用于哔哩哔哩、小红书和抖音“横封面 4:3”，中文 `3:4` 用于抖音“竖封面 3:4”，中文 `16:9` 作为本地通用横版；缺少任一固定封面不得标记当日生产完成。

### 固定格式

- 英文和中文 `16:9` 封面固定为 `1280x720`；中文额外生成 `1200x900`、`4:3` 封面和 `900x1200`、`3:4` 封面。全部使用 sRGB PNG，单张不超过 `2 MB`。
- 英文项目固定输出 `thumbnails/thumbnail.en-US.png`；中文项目固定输出 `thumbnails/thumbnail.zh-CN.png`、`thumbnails/thumbnail.zh-CN.4x3.png` 和 `thumbnails/thumbnail.zh-CN.3x4.png`。每种比例同时保留无文字底图和对应 `thumbnail-spec.<LANG>.<RATIO>.json`。
- 四张封面共享同一视觉语义、主题插画和标题判断。中文 `4:3`、`3:4` 版本允许重新适配插画位置，但不得改变角色、道具、结论或颜色语义。
- `16:9` 固定为左侧约 `58%` 大标题区、右侧约 `42%` 主题插画区；中文 `4:3` 固定为左侧约 `54%` 大标题区、右侧约 `46%` 主题插画区；中文 `3:4` 固定为上方约 `40%` 大标题区、下方约 `60%` 主题插画区。四周保留至少 `32px` 安全边距，标题和角色不得贴边。
- 固定字体使用 `Hiragino Sans GB W6`，英文同样使用该字体以保持字重和几何统一。品牌标签为 `27px`，主标题为 `80-104px`，辅助短句为 `28-30px`。
- 固定色彩为米白网格背景、黑色主标题、Tiny Agent 蓝色标题标记、黄色辅助短句；红色只表示错误、风险或否定。

### 标题与插画

- 主标题只表达本期最重要的冲突或结论，必须在缩小到原图 `20%` 时仍能一眼读出。
- 英文封面标题使用 `3-6` 个单词、`2-3` 行；中文使用 `4-10` 个汉字或一个等价的中英混排短句、`2-3` 行。不得把完整平台标题缩小后塞入封面。
- 封面文案与平台标题互补，不机械重复；平台标题负责完整说明主题和收益，封面负责突出一个判断。
- 插画底图不得生成任何文字、字母、数字或品牌标记。标题必须通过固定排版器叠加，避免生成式图片中的错字和字体漂移。
- 插画保持白板简笔画语义：工程师和 Tiny Agent 必须参与判断或行动，并用一个清楚的对比、路径、风险或结果道具贴合标题含义。
- 固定排版命令使用 `scripts/ai-video-pipeline/create-tiny-agent-longform-thumbnail.mjs`；只允许当期标题、辅助短句和主题插画变化，字体、版式、尺寸和颜色不得随意变化。

### 封面准出

- 自动检查尺寸、色彩空间、文件大小和输出路径。
- 人工查看原尺寸，确认标题无错字、无裁切、无重叠，图文语义一致。
- `16:9` 再检查 `256x144` 缩略预览，中文 `4:3` 再检查 `240x180`，中文 `3:4` 再检查 `180x240`；主标题无法一次读清时必须缩短文案或重新排版。
- 英文发布前必须把英文 `16:9` 封面作为 Postiz 的 YouTube `thumbnail` 媒体随视频一并提交。中文平台准备时，哔哩哔哩与小红书上传中文 `4:3` 封面；抖音分别上传中文 `4:3` 横封面和中文 `3:4` 竖封面，并逐槽位核对预览；中文 `16:9` 继续保留为通用横版。

## 文章分析与内容取舍

### 分析顺序

```text
读取全文与来源元数据
→ 提取中心主旨
→ 提取一级重点、支撑论据、边界和案例
→ 建立知识点依赖关系
→ 按观众价值排序
→ 合并重复观点并删除低价值背景
→ 形成内容地图和章节
→ 分配讲解时长
→ 生成脚本、分镜和动画计划
```

系统必须完整分析文章后再写脚本，不能按段落顺序逐段压缩，也不能按每段字数分配视频时长。

### 内容优先级

| 级别 | 内容 | 处理方式 |
| --- | --- | --- |
| P0 | 中心主旨、关键结论、必要前提 | 必须进入视频并在结尾复述 |
| P1 | 可直接用于判断或实践的方法 | 重点展开，分配主要时长 |
| P2 | 支撑 P0/P1 的机制、边界和代表性案例 | 保留一个最清楚的解释或案例 |
| P3 | 重复论述、低价值背景、罗列性材料 | 合并、压缩或删除 |

来源事实与脚本补充示例必须区分。脚本补充的场景只能帮助理解，不能伪装成原文结论或案例。

## 动态时长规则

### 时长档位

| 内容规模 | 建议时长 | 判断标准 |
| --- | ---: | --- |
| 单一主旨 | `6-9 分钟` | `1-3` 个一级重点，依赖关系简单 |
| 标准文章 | `9-13 分钟` | `4-7` 个一级重点，包含方法、边界或案例 |
| 高密度长文 | `13-18 分钟` | `8` 个以上重点，存在明显依赖、对比或多组案例 |

### 单元分配

- 开头钩子和价值概览合计 `40-60 秒`。
- 中心主旨 `60-90 秒`。
- 每个一级重点 `45-90 秒`。
- 必要案例 `20-40 秒`，同一重点默认只保留一个代表性案例。
- 次要背景 `10-20 秒`，无法支撑核心理解时直接删除。
- 总结和关注引导合计 `40-60 秒`。

最终时长计算：

```text
总时长 = 开头 + 中心主旨 + 一级重点 + 必要机制/案例 + 总结与关注引导
```

计算结果优先落在 `9-13 分钟`。低于 `6 分钟` 时可以保持短片，不得补充重复内容；高于 `18 分钟` 时优先拆成两个独立主题。

脚本文字量只用于估算 TTS，不用于决定内容权重。中文初步估算使用 `220-260 个汉字/分钟`，最终时长以实际 TTS 和字幕时间轴为准。

## 叙事结构

### 开头：钩子与价值概览

- 总时长控制在 `40-60 秒`。
- 先提出反常识、常见损失、关键问题或结果差异。
- 立即说明观众看完能获得的判断、方法或收益。
- 概览只预告中间确实会展开的内容，不罗列无关章节。
- 不做长背景介绍，不连续说欢迎、来源、节目定位和制作说明。

### 中段：按价值展开

- 章节由知识依赖和观众价值决定，不照搬文章目录。
- 每个章节先回答“这是什么”，再回答“什么时候有用”，必要时补充一个例子或边界。
- 最有价值的章节允许占用更多时长；章节不要求等长。
- 一个道具或任务场景默认承载多句话，避免一句话切换一次道具。
- 工程师负责提出问题、交付任务、观察和确认；Tiny Agent 负责搜索、调用工具、执行、保存状态和交付结果。

### 章节边界

- 每个实质内容章节都包含独立的章节开场和本章总结，不能只依赖底部进度条提示切换。
- 章节开场持续 `2.4-3.2 秒`，只展示章节名和一句价值承诺；开场旁白必须自然进入本章内容，不额外加入欢迎语或结构说明。
- 本章总结固定压缩为 `3` 个已讲过的判断、步骤或结论，不引入新知识；时长由实际朗读内容决定，不设置固定秒数。
- 总结必须进入旁白脚本，按“小结。第一，……。第二，……。第三，……。”朗读；画面使用 `1.`、`2.`、`3.`，并按三条旁白的真实 VTT 起点逐条出现。
- 章节开场和总结期间隐藏常驻页眉，保留字幕和全宽章节进度条，形成独立视觉状态。
- 开场价值承诺、本章正文和总结三者必须一一对应。无法在正文中找到依据的承诺或总结项不得展示。
- `前言` 和全片 `总结` 不重复增加章节小结；最后的关注引导持续使用全片总结页，不回切正文画面。

### 结尾：总结与关注引导

- 总结必须复述中间实际讲过的关键点，不引入新知识。
- 优先压缩成 `3-5` 条判断、步骤或一句可复述公式。
- 关注引导只表达持续关注后能获得的价值，例如更快理解 AI/Agent、形成判断、减少踩坑和帮助落地。
- 不把关注理由限制为“解读 Agent 文章”。
- 不承诺下一期的具体主题、案例或发布时间。
- 关注引导与全片总结共用同一视觉页面，只允许在原总结页增加轻量品牌价值提示。

## 章节进度条

### 展示原则

- 进度条固定在画面最底部并横向占满 `1920px` 画布，左右不留外边距；贯穿全片，不进入主画面和字幕区。
- 进度条依次展示 `前言`、实质内容章节和 `总结`；不得把 `钩子`、`内容概览`、`关注引导`、`片头`、`片尾` 作为独立标签。
- 所有标签带连续数字编号，例如 `1. 前言`、`2. 架构判断`、`8. 总结`。
- 每条视频通常展示 `6-9` 个章节，名称控制在 `2-6` 个汉字，编号不计入名称长度。
- 各段宽度按实际章节时长成比例，不使用等宽假进度。
- 当前章节使用深色文字和蓝色进度；已完成章节保留蓝色，未开始章节使用浅灰。
- 章节切换只改变标签状态和进度填充，不做弹跳、放大或遮挡内容的动画。

### HyperFrames 实现

- 进度条是固定 HTML 叠加层，由 HyperFrames/GSAP 根据章节时间轴驱动。
- 进度轨道从画布最左边延伸到最右边，只允许章节之间保留极窄分隔。
- 每个章节声明 `start`、`end`、`label` 和时长权重；这些数据必须从最终旁白和 VTT 生成，不得按脚本文字量估算。
- 每段填充使用 `scaleX: 0 → 1` 和线性缓动，`transform-origin: left center`。
- 进度条、标题和字幕不参与角色或主画面的位移、缩放和姿势切换。
- 章节标签必须来自内容地图，禁止从脚本中的结构标记直接生成。

推荐数据结构：

```json
{
  "chapters": [
    { "label": "前言", "start": 0, "end": 46 },
    { "label": "架构判断", "start": 46, "end": 128 },
    { "label": "基础单元", "start": 128, "end": 190 },
    { "label": "工作流模式", "start": 190, "end": 438 },
    { "label": "自主 Agent", "start": 438, "end": 548 },
    { "label": "工具设计", "start": 548, "end": 625 },
    { "label": "选择方法", "start": 625, "end": 695 },
    { "label": "总结", "start": 695, "end": 730 }
  ]
}
```

## 旁白声音

- 旁白候选使用 `edge-tts` 的中文和英文声音，长期目录维护在 `scripts/ai-video-pipeline/voice-catalogs/edge-tts/`。
- 选型时所有候选使用同一段文本和同一语速试听，避免把文案或语速差异误判为音色差异。
- 中文默认声音为 `zh-CN-YunxiaNeural`，固定使用 `+50%` 语速，即名义 `1.5x`；英文长视频默认声音为 `en-US-AnaNeural`，固定使用 `+30%` 语速。
- 单条视频只使用一个主旁白声音，除非脚本明确存在角色对话；不同章节不随意换声。
- 语速作为模板常量，不允许为单条视频动态快慢调时。实际 TTS 偏离目标范围时，优先增删有价值的讲解、案例或边界，再重新生成旁白和时间轴。
- 准出前必须检查机械断句、中文重音和英文术语发音。

## 横屏构图与动效

- 固定白底和极浅网格，粗黑线条，少量蓝色、黄色和红色语义高亮。
- 顶部只保留 `Tiny Agent` 和当前主题标题，不显示来源、章节编号、解释性注释或黄色标题下划线。
- 标题使用独立标题带。角色、道具、关系线和比较面板的可见区域必须从标题实际底边以下至少 `24px` 开始；视觉重心校正只能整体移动标题带和内容带，不得让内容重新进入标题带。
- 主画面以工程师和 Tiny Agent 为主体，角色必须参与任务，不得长期作为角落里的静态装饰。
- 字幕位于下方独立区域，不显示“核心”“字幕”等标签。
- 底部章节进度条与字幕分层，不得重叠。
- 除标题、字幕和章节标签外，画面文字尽量减少；原文来源放入视频描述。
- 每个正文视觉状态承载 `2-5` 句连续旁白，通常持续 `8-25 秒`；姿势、道具和焦点只在语义变化时切换。超过 `22 秒` 的视觉状态必须设置两次语义焦点变化，但不为制造变化而替换整组道具。
- 每个实质内容章节至少使用 `2` 种构图；时长超过 `60 秒` 的实质内容章节至少使用 `3` 种。同一构图最多连续出现 `2` 次。
- 同一逻辑容器必须先按容器内的元素数量排版，不能直接套用全场元素坐标：`1` 个图标居中；`2` 个图标等尺寸、同一水平线并围绕容器中心对称；无方向语义的 `3` 个图标以容器中心为重心形成等边三角形；`4` 个图标使用规则 `2×2` 网格。链路、分支和循环分别使用横向链、树形和环形几何，不套用数量模板。
- 对角线散点只用于明确表达方向、递进、冲突或因果关系，并且必须有连线或动作承接；没有这类语义时，不得用左上加右下的方式填充空白。
- 正文构图统一经过“逻辑分组 → 语义几何 → 安全区约束 → 可见像素视觉重心校正”四步。主体安全区为 `x=56-1864`、`y=120-820`；角色、道具、标题和比较面板作为一个整体平移，不单独补偿某个元素。
- 正文视觉重心目标为 `(960, 470)`。编译期按透明素材的可见边界和标题字重估算视觉质量，水平偏移不得超过 `160 px`、垂直偏移不得超过 `100 px`；渲染抽帧按实际像素复核，水平偏移不得超过 `150 px`、垂直偏移不得超过 `100 px`。
- 角色位置和大小随构图调度：流程图可退到两侧参与交付，重点页放大 Tiny Agent，主结论页可让角色和结论形成主次关系。可镜像的构图应交替视觉轴线，角色不能全片固定在同一坐标。
- 每 `6-10 秒` 至少出现一次与旁白语义一致的视觉节拍，包括主体聚焦、关系线变化或角色姿势变化；不得为了满足频率逐句换页。
- 重点内容只对当前角色、道具或关系执行 `1.03-1.10` 倍的短暂聚焦，或直接切换到大标题/大主体构图；不得缩放整个场景制造强调。
- 不做持续镜头推进、频繁缩放和逐句翻页。视觉变化由构图切换、角色调度、重点聚焦和道具状态共同完成。
- 角色姿势使用短位移或短交叉淡化；道具优先留场，通过描边、连接关系和局部缩放改变焦点。
- 流程连接线必须从一个图标的可见边界连接到另一个图标的可见边界，不得穿过透明区域连接中心点；无明确语义时不额外添加焦点圆环。
- 复杂流程先尝试转化为角色行动。明确的比较、步骤、分类或判断结构可以使用简洁的对比、流程和网格构图，但不得退化成满屏文字课件。

### 时间轴优先编排

- 固定顺序为：分析来源并完成脚本 → 生成最终 TTS 与 VTT → 建立段落和字幕时间刻度 → 将场景、角色、道具和重点动画放入对应时间段 → 编译视频。
- 双语版本分别执行完整时间轴流程。英文版和中文版不得共享 TTS、VTT、`timing-map.json` 或 `animation-plan.json`；只能共享内容地图、章节语义、角色任务和构图意图。
- 英文或中文脚本为适应自然表达允许调整句序和局部例子，但中心主旨、内容优先级、章节承诺、可复用产物和事实边界必须一致。
- `scene-plan.json` 只负责声明脚本段落与视觉语义的对应关系，不自行估算时长；`timing-map.json` 记录段落、场景、道具、小结和关注引导的真实时间。
- 场景边界必须落在 VTT 句子或脚本段落边界。道具优先在对应关键词所在字幕开始时出现；没有直接关键词时，才使用该场景真实字幕提示点的语义分位。
- 每条视频必须生成 `animation-plan.json`。每个道具只能归属一个语义节拍，节拍必须绑定最终旁白中真实存在的短语；编译器从最终字幕时间轴计算出现时刻，禁止根据道具序号、字幕数量或场景时长均分。
- 同一短语共同表达一个概念的道具作为一组同时出现；只有旁白明确讲述先后步骤时才逐项出现。列表在同一句中快速读出、剩余展示时间不足或逐项出现没有新增含义时，必须合并为一个节拍。
- 每个语义节拍出现后至少保留 `1 秒`可读时间。触发短语过晚时，应提前合并到上一个完整语义组或调整场景边界，不得把动画强行挤在转场前。
- 不允许先按平均时长铺画面，再让旁白迁就画面。脚本或旁白变化后，必须重新生成 VTT、`animation-plan.json` 和全部视觉时间锚点。

### 构图类型

| 类型 | 适用内容 | 画面规则 |
| --- | --- | --- |
| `hero` | 章节主结论、反常识判断 | 结论放大，角色与一个主道具形成强主次 |
| `process` | 顺序、循环、工具调用 | 横向或环形连接，角色位于流程起点和结果端 |
| `comparison` | 方案差异、适用边界 | 左右双区分别计算布局；每区 `1` 个证据居中、`2` 个证据水平并列，不跨区使用对角坐标 |
| `grid` | 分类、问题清单、并列能力 | `2-5` 个单元形成稳定网格，角色负责指向或确认 |
| `focus` | 风险、结论、记忆点 | 主体明显放大，其他信息降级为辅助证据 |

### 转场规则

- 同构图连续场景使用 `0.45-0.50 秒`的短距离推移与遮罩揭示；不同构图使用 `0.50-0.55 秒`的遮罩揭示；章节切换使用 `0.70-0.80 秒`的纵向覆盖揭示。
- 转场以一个主运动为准，禁止旧画面和新画面分别渐出渐入。新画面覆盖完成后必须立即结束旧场景生命周期。
- 章节开场向正文使用遮罩揭示，章节正文向总结使用侧向揭示；不使用全屏黑场或白场闪切。
- 字幕先在 `0.1 秒`内退出，下一条再用约 `0.18 秒`进入，两个字幕文本不得同时可读。
- 转场前后主体位置需要有连续关系；只有章节边界允许明显改变视觉轴线。
- 正文转场不得反复让整张画面滑出画布；强调动画不得带动字幕、进度条或整个场景缩放。
- 新场景中的角色、道具和标题必须从遮罩揭示起点开始入场，禁止先完整出现再执行入场动画。
- 新场景揭示前 `0.02 秒`必须先写入角色、标题、比较面板和章节卡的隐藏起始状态，再执行单向进入动画；不得使用延迟启动且 `immediateRender:false` 的 `fromTo` 让元素先以最终状态暴露。
- 角色入场只使用 `8-10px`短上浮和透明度变化，持续约 `0.56 秒`；场景遮罩已经承担主运动时，不再叠加角色缩放。人物与 Tiny Agent 可错开不超过 `0.04 秒`，但都必须在画面揭示前完成状态预置。

- 标题只承载一个承诺，使用“问题/冲突 + 明确收益或主题”的结构；不得连续堆放多个概念、公司名、型号或热门词。

标题候选按以下维度自动评分，任一候选存在事实夸大或承诺未被脚本兑现时直接淘汰：

| 维度 | 权重 | 判断 |
| --- | ---: | --- |
| 点击吸引力 | `35%` | 是否提出目标观众正在经历的问题、损失或结果差异 |
| 搜索相关性 | `30%` | 主搜索词是否自然出现，是否符合真实搜索表达 |

| 内容兑现度 | `15%` | 开头和正文是否完整兑现标题承诺 |

`content-map.json` 记录来源、核心主旨、内容优先级、删除项、章节和进度条标签。两个语言项目分别记录自己的时间分配、脚本、TTS、VTT、时间轴和动画计划。`publish-metadata.en-US.json` 记录英文发布元数据，`publish-metadata.zh-CN.json` 记录中文本地交付元数据。两个版本的 `summary.json` 分别记录最终脚本、音视频时长、章节和全部准出结果。

## 准出检查

### 内容与脚本

- 已分析完整来源，不是逐段缩写或按文字量分配时长。
- 中心主旨、P0 和 P1 内容全部进入视频；删除项有明确原因。
- 开头在 `40-60 秒`内完成钩子和价值概览，且与中间章节完全对应。
- 总结只复述中间讲过的关键点，并提供可记忆的判断、步骤或公式。
- 关注引导表达 AI/Agent 内容的持续用户价值，不限定为文章解读，不承诺下一期具体内容。
- 所有来源结论可回溯；补充示例没有被描述成来源事实。
- 目标时长符合内容档位，最终 TTS 落在 `6-18 分钟`内，默认优先 `9-13 分钟`。
- 发布标题的核心问题与开头钩子一致，标题承诺在正文中得到完整兑现，没有原文标题直译、关键词堆砌或误导表达。
- 简介前两行包含当期主关键词和明确收益；英文 YouTube 简介包含三点收益、真实章节、来源、追踪链接、关注价值和 `4-6` 个相关 hashtag；中文平台简介包含三点收益、来源和关注价值，且没有章节标题或时间戳。抖音与小红书的话题必须绑定为原生实体；哔哩哔哩的简介不得保留 `#话题` 文本，投稿 4:3 与个人空间 16:9 两个封面槽位均已设置并验证。
- 英文和中文版本的脚本、旁白、字幕、章节时间和动画均独立对齐；不存在跨语言复用时间戳。
- `publish-metadata.en-US.json` 与英文最终视频、章节时间、来源和英文 `summary.json` 一致；`publish-metadata.zh-CN.json` 与中文本地视频及中文 `summary.json` 一致。

### 角色、画面与进度条

- 工程师和 Tiny Agent 外形稳定，姿势交换使用统一锚点，没有位置跳动。
- 角色承担讲解动作，主画面不是静态角色加 PPT 内容。
- 每个主要道具至少承载一组连续旁白，不逐句更换道具。
- 每个实质内容章节存在章节开场和可朗读的三点编号总结；开场承诺、正文重点和总结内容一致。
- 每个实质内容章节满足构图多样性要求，没有超过 `2` 个连续同构图场景。
- 重点内容存在明确的尺度或构图调度，不依赖常驻文字重复强调。
- 画面没有来源标签、解释性杂项、黄色标题下划线或字幕标签；章节编号只出现在章节开场和底部进度条。
- 章节进度条包含 `前言`、`6-9` 个实质内容段和 `总结`，标签全部带连续数字编号。
- 进度条不出现“钩子”“概览”“关注引导”“片头”“片尾”。
- 各章节宽度与实际时长成比例，当前、完成和未开始状态清楚。
- 进度条、字幕、角色、标题和关键道具没有重叠或越界。
- 所有正文场景通过标题带隔离校验，标题底边与任何角色、道具、关系线或比较面板之间至少保留 `24px`。
- 每个逻辑容器中的双图标保持等尺寸、同一水平线、间距一致并围绕容器中心对称；编译期布局校验通过。
- 无方向语义的三图标组形成等边三角形；全部正文场景通过安全区、视觉重心和实际像素平衡校验，运行 `pnpm run check:balance` 后失败场景数为 `0`。
- `animation-plan.json` 覆盖全部正文道具；运行 `pnpm run check:semantics` 后语义兜底数、重复绑定数、未绑定数和标题带失败场景数均为 `0`。
- 运行 `pnpm run check:transitions` 后，全部角色入场均在内容揭示前完成状态预置，延迟角色 `fromTo` 数和失败角色数均为 `0`。
- 所有流程连接线都止于图标可见边界，没有穿过图标中心、文字标签或角色。
- 进度轨道横向覆盖完整画布，首段从 `x=0` 开始，末段在 `x=1920` 结束。
- 章节开场和总结期间常驻页眉隐藏，字幕与进度条仍可读。
- 场景切换没有双重渐隐、短暂黑场、旧画面残留、角色先出现后回跳或字幕双行叠加。

### 音视频与 HyperFrames

- 成片为 `1920x1080`、`30fps`，包含 H.264 视频和 AAC 音频。
- 音频清楚，无削波、爆音、明显机械断句或静音尾巴；目标综合响度为 `-16 至 -18 LUFS`。
- 字幕与旁白同步，单屏不超过两行，长句按语义换行。
- `timing-map.json` 证明场景、小结条目和道具节拍来自最终 VTT，而不是字数比例。
- `pnpm dlx hyperframes check . --snapshots --frame-check=severity=error` 通过。
- 在开头、每个章节中点、章节切换点和结尾抽帧，确认无黑帧、空白主画面、角色跳动、文字溢出或进度错误。
- 最终 MP4 存在且大小合理，`ffprobe` 的分辨率、帧率、时长和音视频流符合规则。
- 任一准出项失败时停止发布，修复源脚本、资产、时间轴或布局后重新生成。
