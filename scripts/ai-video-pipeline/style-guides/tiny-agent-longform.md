# Tiny Agent 横屏长视频生产基线

## 用途

这份文档是 Tiny Agent 横屏讲解视频的长期生产基线。输入为一篇文章或一组同源资料，输出为结构统一、时长自适应、可自动检查的完整讲解视频。

长视频不按原文段落或文字量平均分配篇幅。系统必须先理解全文，再围绕核心主旨、关键方法、重要边界和高价值案例重新组织内容。成片采用总分总结构，让观众在开头获得明确收益预期，在结尾重新记住中间讲过的关键点。

当前深度内容档必须同时读取并执行 [`tiny-agent-deep-longform-cognitive-load.md`](tiny-agent-deep-longform-cognitive-load.md)。该文件记录已确认的双语故事化叙事、固定语速、稳定画面节拍、自然章节衔接和首句全屏动效规则；与本文历史时长、语速、逐章口播小结或开头秒点冲突时，以该文件为准。

短视频继续遵循 `scripts/ai-video-pipeline/style-guides/agent-sketchbook.md`，两套规则互不覆盖。

## 输出定位

- 平台：英文版为 YouTube 横屏长视频；中文版只生成本地成片、通用中文元数据和封面素材，不再代填任何中文平台上传页。
- 日更任务固定同时输出英文版和中文版。两个版本复用相同内容地图、内容优先级、章节结构和视觉语义，但分别使用自然英文和自然中文脚本。
- 画布：`1920x1080`，`16:9`，`30fps`。
- 编码：H.264 视频、AAC 音频。
- 当前深度内容档固定成片范围：`9-12 分钟`。时长是内容编辑后的结果，不得为了补时保留枯燥长句、重复解释或低价值背景。
- 超过 `12 分钟` 时先删除确认不增加事实、证据、判断、边界、步骤或理解帮助的内容；仍超过时拆成两个独立主题。不得删除权威来源、必要证据、关键边界或代表性案例来机械压时。
- 风格：白板简笔画、角色驱动、低文字密度；角色演绎与标题页、流程、对比、网格和重点聚焦等信息构图交替使用，避免单一左右构图和纯课件翻页。

## 固定主题色

- 长视频固定采用配色方案 `C - 中性纸灰`；基础画布色为 `#ECECEA`。
- `#ECECEA` 统一用于画布、卡片/面板、字幕框、工程师浅色衣服、Tiny Agent 外壳与手脚、无语义道具主体和封面中性底色。
- 主墨色固定为 `#111413`，用于标题、正文、粗线轮廓和主要信息。
- 不再使用纯白 `#FFFFFF` 或近白色为卡片、字幕框、角色与道具另建一套中性层级；层次优先依靠黑色描边、留白、间距和低对比阴影。
- 蓝、黄、红只用于明确语义；底部章节进度允许使用固定浅蓝 `#A8D8F0` 表示已播放，其余导航状态不得为了装饰单独增加蓝色。
- 图片生成提示词、HTML/CSS 主题变量、封面排版器和视觉准出都必须显式使用 `#ECECEA`，不得回退到历史白底或米白底。

## 留存优先硬门槛

以下规则对英文版和中文版同时生效。两种语言共享同一留存结构和视觉语义，但必须根据各自最终 TTS/VTT 独立计算时间，不能复用另一语言的秒点。

### 首句逐字动效问题卡

- 中英文开头永久采用已确认样片 `var/hyperframes-showcases/2026-07-22-tiny-agent-hook-type-on-sample-zh-CN/renders/tiny-agent-hook-type-on-sample.zh-CN.mp4` 的“纸灰手绘 + 花式字体 + 语音同步逐字弹入”方案。正式封面不得进入视频开头；不得回退为首帧一次性展示完整问句的静态大字卡，也不得改用漫画爆破、赛博霓虹或其它候选样片方向。
- `0 秒`直接进入问题动效场景，使用 `#ECECEA` 纸灰网格背景、`#111413` 主墨色和蓝/黄/红语义强调。允许复用确认样片中的顶部手绘线、淡色问号、语音进度线和 Tiny Agent 入场，但第一视觉主体始终是正在随旁白出现的问题文字；问题场景期间隐藏正式字幕框、常驻页眉和章节进度条。
- 第一句话必须在 `5 秒`内念完，是一个从观众角度提出、直接呼应发布标题和本期核心问题的问句。中文必须自然包含 `AI 智能体` 或 `AI Agent`，英文必须自然包含 `AI Agent` 或 `AI Agents`；不得用泛化口号或先介绍频道。
- 中文首句采用“你知道怎么……吗？”这类直接问句，例如：`你知道怎么给 AI 智能体高信号上下文吗？`。英文使用自然的第二人称问句，例如：`Do you know how to give an AI Agent high-signal context?`，不机械逐词翻译中文。
- 必须先基于对应语言最终 TTS/VTT 建立首句字级时间表，再编排动画。中文每个汉字和标点、英文每个单词内字母在对应语音触发点前 `0.10-0.15 秒`开始弹入，让观众先看到再听到；不得提前展示后续完整单词或整句。中英文不得复用时间戳。
- 每个字形使用确认样片的短促弹入语言：透明度从 `0` 到 `1`，同时配合小幅位移、旋转和缩放回弹；相邻字形交替改变入场方向或旋转方向，避免整行统一淡入。`AI` / `AI Agent`、核心关键词和句末问号可以使用蓝色、黄色标记或红色爆发线加强，但效果必须短促、确定性、可 seek，不能遮挡文字。
- 中文优先嵌入确认样片使用的 `ZCOOL KuaiLe` 花式字体；英文使用可嵌入、粗重、略带手绘不规则感且易读的展示字体。两种语言必须保持相同的字号层级、黑色主墨、强调色和弹入节奏，不能让英文版退化成普通无衬线静态字幕。问题排版优先 `2` 行，英文过长时最多 `3` 行；按语义短语断行，中文不得出现单字或标点孤行，英文不得拆词或把冠词、介词单独留在一行。
- 问题文字在 `1920x1080` 画布中央安全区内使用可容纳的最大字号，不为 Tiny Agent 或装饰预留大片空白。完整问题的可见包围盒应尽量覆盖主内容安全区，建议宽度不低于画布 `82%`、高度不低于画布 `52%`；Tiny Agent 应缩小或移位避让文字。以真实浏览器 DOM 包围盒检查每个字形、强调标记、问号爆发线和 Tiny Agent 均无裁切、无溢出、无非预期遮挡。
- 句末问号按确认样片使用多段关键帧完成“放大冲入 → 轻微反向回弹 → 稳定”的收束，并可同时触发短暂放射线。最后一个字形稳定、问句念完后，完整问题必须继续保持可读 `1.2-1.6 秒`再硬切正文；停留期间不额外插入新文案、长静音或黑帧。
- 首句问题卡和紧随其后的内容不加入关注、收藏或保存引导。第一句问完后直接进入权威背书与本期结论；关注收藏画面和文案统一后移到本节规定的“前言”最后部分。
- 问题卡切走后立即显示底部章节进度条；不得延迟到第二个章节或第一个正文场景结束。

### 前 30 秒

- `12 秒`前必须自然说出当期权威来源或研究机构，并给出一个明确结论；只展示机构名和核心判断，不展示长来源标题、URL 或来源卡片墙。
- `20 秒`前必须说清观众当前可能遭受的具体损失，并承诺本期能获得的判断、方法或工具。
- `30 秒`前必须预览最终可复用产物并开始实质解释，不得把前 `30 秒`全部用于欢迎语、节目定位或文章背景。
- 开头留存段通常为 `30-45 秒`。前 `30 秒`句间普通停顿不得超过 `0.2 秒`；正文普通句界、知识单元和章节边界分别使用约 `0.25/0.5/0.8 秒`的固定处理停顿。

### 前言末尾关注收藏

- 原关注收藏画面和文本不删除，固定放在“前言”章节的最后一个场景：必须位于权威背书、具体损失、观众收益、最终工具预览和实质解释之后，并在第一个实质内容章节开场之前。不得提前回到首句问题卡之后或前 `30 秒`的内容门槛之前。
- 中文旁白固定完整念出：`视频全程干货高价值，让你成为更擅长使用 AI 的人，内容较长，点点关注收藏不迷路。`。英文固定完整念出：`This video is packed with practical, high-value insights to help you get better at using AI. It's a longer one, so follow Tiny Agent and save it so you don't lose it.`。两版不得删减、改写或拆散到其它章节。
- 画面复用已确认 v5 `promise` 场景的语义和视觉结构：使用纸灰背景、人物与 Tiny Agent 共同参与、一个收藏/保存语义道具，以及大号粗体价值文案；不得改成发布封面、普通字幕独占画面或片尾关注结束页。可按当期语言和安全区重新排版，但画面与对应旁白必须在同一 VTT 时间段完整出现。
- 该场景仍属于 `1. 前言`，底部章节进度条保持显示并处于前言段；不得新增“关注”“收藏”或 CTA 章节标签。场景结束后的下一帧进入第一个实质章节开场。

## 双语封面

每期必须与双语成片同步生成四张封面：英文 YouTube `16:9` 一张；中文固定 `16:9`、`4:3`、`3:4` 三张。英文封面用于 YouTube 发布，三张中文封面只作为本地素材交付，自动化不上传到中文平台。不再生成 `thumbnail.zh-CN.youtube-style.png` 或其它备选中文封面；缺少任一固定封面不得标记当日生产完成。

中文三比例封面的固定视觉参考为 [`references/tiny-agent-chinese-cover-three-ratio-reference.png`](references/tiny-agent-chinese-cover-three-ratio-reference.png)。每次写封面规格、生成 SVG/PNG 和执行 QA 前都必须查看这张参考图，并同时读取本节当前规则；参考图负责锁定整体风格、文字主导关系和三种比例的构图差异，本节文字规则负责锁定尺寸、占比和准出门槛。

### 固定格式

- 英文 YouTube `16:9` 发布母版固定为 `3840x2160`；中文 `16:9` 固定为 `1280x720`，中文 `4:3` 固定为 `1200x900`，中文 `3:4` 固定为 `900x1200`。全部使用 sRGB PNG，单张不超过 `2 MB`。
- 英文项目固定输出 4K 发布母版 `thumbnails/thumbnail.en-US.png`，并生成 `thumbnail.en-US.1280x720.png` 与 `thumbnail.en-US.256x144.png` 作为 QA 预览；发布时只能提交 4K 母版。中文项目只输出 `thumbnails/thumbnail.zh-CN.png`、`thumbnails/thumbnail.zh-CN.4x3.png`、`thumbnails/thumbnail.zh-CN.3x4.png`。每种比例同时保留无文字底图与对应 `thumbnail-spec.<LANG>.<RATIO>.json`。
- 四张封面共享同一主题语义、核心结论和角色身份。英文和中文全部比例都以完整插画作为底图，只确定性叠加标题与短装饰线；不得再用整块不透明或近乎不透明的左侧面板遮住插画。不同语言和比例允许重新适配文字与插画位置，但不得改变结论或颜色语义。
- 英文 YouTube `16:9` 封面固定采用“大标题 + 大主体 + 少信息”的 `title-hero` 结构，不使用四分图、等分网格、缩小版流程图或多卡片信息墙。即使正文包含四个或更多并列模块，也必须压缩成一个统一主视觉，例如四向选择器、四瓣技能轮、单张判断表或一个核心结果物体。
- 中文三张封面全部复用文字主导的 `title-hero` 规则：文字获得主要视觉预算，完整插画保持可见，标题按比例自适应为 `2-4` 行。中文 `16:9` 固定使用大标题左侧、完整插画右侧构图；中文 `4:3` 使用大标题左侧、完整插画右侧构图，且可见标题包围盒面积必须至少占画布 `50%`；中文 `3:4` 使用下述独立分区门槛。所有包围盒通过只保留对应图层的原尺寸渲染实测，不得用预估区域替代。不得用面板、蒙层或矩形色块整体截断插画；角色与核心物体仍需一眼可识别。
- 中文 `3:4` 永久固定为上文字、下插画的 `60/40` 纵向分区，不再使用候选槽或根据局部空白改变插画位置。黄色分割线中心固定在 `y=0.60H`；顶部 `60%` 只放标题，标题使用自然中文 `2-4` 行，在不越过分割线的前提下自动放大并尽量铺满，实测可见标题包围盒至少占文字区面积 `68%`。标题过长时先调整断行或精简文案，不得侵入下方图片区。
- 中文 `3:4` 底部 `40%` 只放完整插画。插画使用 `contain` 等比缩放、水平居中，在图片区四周保留至少 `16px` 安全边距；角色、天线、核心物体、工具和阴影都必须完整显示，不得裁切、拉伸或与标题/分割线重叠。插画应在满足完整显示和安全边距后尽可能放大。
- 英文、中文 `16:9` 与中文 `4:3` 可以根据标题长度微调左右占比，但必须保持一个标题区、一个主角色和一个核心物体，不得改变参考图中的文字左、插画右关系。四周保留至少 `32px` 安全边距，标题和角色不得贴边。
- 固定字重使用 `Hiragino Sans GB W6` 或几何一致的 `Arial Black`。标题型封面的英文主标题按长度在约 `96-160px` 间自适应，关键词优先保持 `124-160px`；中文使用约 `108-156px`。标题必须是画面第一视觉层级，不用小字号完整复述平台标题。
- 固定色彩为 `#ECECEA` 中性纸灰网格背景、`#111413` 主标题、Tiny Agent 蓝色标题标记、黄色辅助短句；红色只表示错误、风险或否定。

### 标题与插画

- 主标题只表达本期最重要的冲突或结论，必须在缩小到原图 `20%` 时仍能一眼读出。
- Tiny Agent 长视频的封面文案必须先让陌生观众识别这是 AI Agent / 智能体主题，再表达当期判断。英文封面标题必须自然包含 `AI Agent` 或 `AI Agents`；中文 `16:9`、`4:3`、`3:4` 三张封面标题必须自然包含 `AI Agent` 或 `智能体`。不得只使用“越顺眼越要检查”“别只收集资料，要支持决策”这类脱离上下文后看不出 AI / Agent 主题的口号；应改为“AI 智能体输出越顺眼越要检查”“AI Agent 调研要支持决策”或“交给智能体前先写清六件事”这类同时交代对象与主题的表达。
- “少信息”指只表达一个中心判断，不代表限制成三个单词。英文封面优先使用自然、完整的 `4-8` 词短句，必要时允许 `2-9` 词，按 `2-4` 行排版；通过字号层级突出关键词，其余词仍须清晰可读。中文使用 `4-14` 个汉字或一个等价的中英混排短句、`2-4` 行。不得为了追求极少字数把主题压缩成含义模糊的口号，也不得把完整平台标题等比缩小后塞入封面。
- 封面文案与平台标题互补，不机械重复；两者都必须保留 AI Agent / 智能体主题身份，平台标题负责完整说明主题和收益，封面负责突出一个判断。
- 插画底图不得生成任何文字、字母、数字或品牌标记。标题必须通过确定性排版叠加，避免生成式图片中的错字和字体漂移。
- 封面插画每期根据内容从零设计为一幅完整场景。图片生成时必须同时引用活跃 Tiny Agent 资产包的角色图或角色 contact sheet 作为身份参考，但允许为整体观感调整姿势、比例、线条和局部细节；只要保留圆润白色机器人、黑色脸屏、蓝色眼睛与天线、棕色工具腰带、红蓝工具等核心识别锚点，不要求逐像素复刻单张 PNG。
- 角色、道具、手势、阴影和透视必须在同一张插画中原生生成并形成真实接触关系，不再默认把透明角色 PNG 直接贴到独立生成的背景上。整体插画完成度、视觉冲击和语义清晰度优先于角色素材的细节完全一致。
- 插画保持白板简笔画语义：角色必须参与判断或行动，并用一个清楚的对比、路径、风险或结果道具贴合标题含义。品牌统一依靠纸灰网格、主墨色、蓝黄语义色、粗线条、稳定的角色识别锚点和大号粗体，而不是依靠重复同一版式。
- 英文封面只保留一个主角色和一个核心流程/物体。文字是第一主体，标题区获得约 `60%-70%` 的视觉预算；右侧插画的实际可见包围盒只占整张画面约 `30%-40%`，不得达到或超过半幅。可通过缩小、右移、裁掉非核心留白来控制插画占比，但角色与核心物体必须完整可辨。正文中的多个并列概念必须合并进同一个选择器、轮盘、卡片或结果物体，不得拆成多个小角色或小面板。不满足该比例或在 `256x144` 下无法一眼识别标题与主视觉时不得准出。
- 封面遵循“少而大”：除标题外默认只保留 `1` 个主角色、`1` 个核心物体或最多 `3` 个必要关系节点；不得把正文流程图、完整判断表或多组小道具整体缩进封面。

## 文章分析与内容取舍

### 分析顺序

```text
读取全文与来源元数据
→ 提取中心主旨
→ 提取一级重点、支撑论据、边界和案例
→ 建立知识点依赖关系
→ 按观众价值排序
→ 合并重复观点并删除低价值背景
→ 形成内容地图和章节
→ 分配讲解时长
→ 生成脚本、分镜和动画计划
```

系统必须完整分析文章后再写脚本，不能按段落顺序逐段压缩，也不能按每段字数分配视频时长。

### 内容优先级

| 级别 | 内容                                | 处理方式                   |
| ---- | ----------------------------------- | -------------------------- |
| P0   | 中心主旨、关键结论、必要前提        | 必须进入视频并在结尾复述   |
| P1   | 可直接用于判断或实践的方法          | 重点展开，分配主要时长     |
| P2   | 支撑 P0/P1 的来源证据、机制、边界和代表性案例 | 按理解与可信度价值保留必要内容 |
| P3   | 重复论述、低价值背景、罗列性材料    | 合并、压缩或删除           |

来源事实与脚本补充示例必须区分。脚本补充的场景只能帮助理解，不能伪装成原文结论或案例。

生成双语脚本前必须执行一次独立的内容编辑检查。目标是提高信息密度和表达质量，不是为了命中时长而机械删减：

- P0 和 P1 必须保留；P2 按证据价值判断，凡是能建立来源可信度、支撑结论、解释机制、限定适用边界或帮助观众迁移使用的内容都应保留，不设置固定数量上限。
- 权威机构、原文或研究的明确归属是内容证据，不得默认视为低价值背景。脚本可以在能增强可信度的位置自然说明来源；只删除与结论无关的机构历史、发布过程或元信息堆砌。公开视频简介是否展示来源继续遵循独立的发布文案规则。
- 只删除确认没有增加新事实、证据、判断、边界、步骤、必要例证或理解帮助的重复表达、过渡性客套和相似案例；不得仅因为预计超出优先时长就删除高价值内容。
- 同一观点应避免原句式重复，但在开头建立可信度、正文解释和总结回收时允许从不同角度再次表达，只要每次承担不同叙事功能。
- 英文和中文分别按自然表达精简，不能因为机械翻译而保留另一种语言中的赘词、重复主语或过长连接句。

## 动态时长规则

### 时长档位

| 内容规模   |   建议时长 | 判断标准                                             |
| ---------- | ---------: | ---------------------------------------------------- |
| 单一主旨   | `9-10 分钟` | 一条日常故事、一个核心机制和一个可复用工具           |
| 标准文章   | `10-11 分钟` | 机制、诊断、行动三段递进，包含必要证据、边界和案例   |
| 高密度长文 | `11-12 分钟` | 只保留一个完整问题链；无法在 `12 分钟`讲清时拆成两期 |

### 单元分配

- 开头留存段和价值概览合计 `30-45 秒`，且 `30 秒`前开始实质解释；关注收藏引导只占用前言最后一个场景，不得挤占前 `30 秒`的内容门槛。
- 中心主旨和故事冲突 `60-90 秒`。
- 每个实质模块 `120-180 秒`，按机制、诊断、行动的知识依赖自然分配。
- 核心故事贯穿实质模块，不作为正文之外的独立补时段。
- 次要背景最多 `10 秒`，无法支撑核心理解时直接删除。
- 总结和关注引导合计 `45-70 秒`。

最终时长计算：

```text
总时长 = 开头 + 中心主旨 + 一级重点 + 必要机制/案例 + 总结与关注引导
```

计算结果必须落在 `9-12 分钟`。不足 `9 分钟` 的内容不得靠重复解释补时，应补充真正支撑 P0/P1 的必要机制、边界或案例；超过 `12 分钟` 时先确认是否可以删除没有信息增量的重复内容，不能删除必要证据、边界或案例；仍超出时拆成两个独立主题。

脚本文字量只用于估算 TTS，不用于决定内容权重。中英文都必须以固定 `+10%` 语速生成的实际 TTS 和字幕时间轴为最终时长依据。

## 叙事结构

### 开头：钩子与价值概览

- 总时长控制在 `30-45 秒`，严格执行“全屏问题大字卡 → 权威背书 → 具体损失 → 最终工具预览 → 实质解释 → 前言末尾关注收藏”的留存顺序。
- 权威来源用于建立可信度，不以“文章解读”为卖点；机构名和核心结论必须在 `12 秒`前出现。
- 概览只预告中间确实会展开的内容，不罗列无关章节；可复用产物必须在 `30 秒`前可见。
- 不做长背景介绍，不连续说欢迎、节目定位和制作说明。

### 中段：按价值展开

- 章节由知识依赖和观众价值决定，不照搬文章目录。
- 实质内容优先收束为不超过三个连续模块：机制、诊断和行动。每段都承接前一段尚未解决的问题，不重新播报目录。
- 每个模块先从同一条日常故事提出问题，再解释“为什么”“怎样判断”和“如何行动”；抽象规则在约 `20 秒`内回到案例。
- 陌生技术术语首次出现时，先用一句普通人能理解的话说明它是什么、用来判断什么，再给数据、定义或边界。
- 最有价值的章节允许占用更多时长；章节不要求等长。
- 一个道具或任务场景默认承载多句话，避免一句话切换一次道具。
- 工程师负责提出问题、交付任务、观察和确认；Tiny Agent 负责搜索、调用工具、执行、保存状态和交付结果。

### 章节边界

- 每个实质内容章节包含独立章节开场，但不设置重复正文的固定三点口播小结。
- 章节开场只展示章节名和一句价值承诺；开场旁白必须自然进入本章内容，不额外加入欢迎语或结构说明。章节名和价值承诺使用至少为字幕字号 `130%` 的粗体，中文与英文都不得退回小号辅助标签。
- 模块结尾使用一句因果收束或一个尚未解决的问题自然进入下一模块。不得朗读“本章小节”或 `Chapter recap`，也不得把三个正文结论换一种说法再念一遍。
- 转场画面可以保留一个上一段已确认的判断和一个下一段问题，但不得形成新的信息墙；文字至少为字幕字号 `130%` 且加粗。
- 章节开场和自然转场期间隐藏常驻页眉，保留字幕和全宽章节进度条。
- 开场价值承诺、正文和自然转场必须形成同一因果链。无法在正文中找到依据的承诺或收束判断不得展示。
- 完整知识回收只放在全片 `总结`；总结必须用当期可复用工具重新解决开头故事。总结结束后固定硬切到独立关注结束页，不回切正文画面。

### 结尾：总结与关注引导

- 总结必须复述中间实际讲过的关键点，不引入新知识。
- 优先压缩成 `3-5` 条判断、步骤或一句可复述公式。
- 关注引导不得根据当期主题改写。中文旁白永久固定并必须逐字念出：`关注 Tiny Agent，成为更擅长使用 AI 的人！`，不得改成两句，也不得追加其它内容。
- 英文旁白必须逐字念出：`Follow Tiny Agent. Tiny Agent helps you get better at using AI.`，不得省略第一句，也不得在第二句后追加内容。
- 固定关注句同时用于对应语言的旁白结尾和发布简介，不把关注理由限制为“解读 Agent 文章”。
- 不承诺下一期的具体主题、案例或发布时间。
- 当 `Follow Tiny Agent`（中文为“关注 Tiny Agent”）开始发音的第一帧，必须从全片总结页硬切到独立结束页，不使用淡入淡出。
- 独立结束页隐藏当期内容标题、常驻页眉、总结条目、字幕框和章节进度条；左侧或中央使用一个大幅笑脸/挥手 Tiny Agent 图片。英文版另一侧必须完整显示 `Follow Tiny Agent` 和 `Tiny Agent helps you get better at using AI.`；中文版同一位置必须完整显示 `关注 Tiny Agent` 和 `成为更擅长使用 AI 的人！`。两版使用相同角色图、构图、配色和动效，只替换对应语言的文案与音轨；不得出现来源、章节、下一期预告或其它正文信息。
- 结束页使用与正文一致的 `#ECECEA` 中性纸灰、`#111413` 主墨色与 `#117ABD` 蓝色强调；固定 CTA 语音结束后只允许保留 `0.15-0.35 秒` 画面余量，不得产生明显静音尾巴。

## 章节进度条

### 展示原则

- 进度条固定在画面最底部并横向占满 `1920px` 画布，整体高度固定为 `52px`，左右和底部不留外边距；全屏问题卡与第一句结束后立即出现并贯穿全片总结，进入独立关注结束页时隐藏，不进入主画面和字幕区。
- 进度条依次展示 `前言`、实质内容章节和 `总结`；不得把 `钩子`、`内容概览`、`关注引导`、`片头`、`片尾` 作为独立标签。
- 标签固定添加从 `1` 开始的阿拉伯数字、英文句点和一个空格，例如 `1. 前言`、`2. 架构判断`、`3. 总结`。英文版使用相同格式，例如 `1. Introduction`。编号按最终章节顺序连续递增，不跳号、不使用前导零。
- 每条视频通常展示 `6-9` 个章节；中文名称控制在 `2-6` 个汉字，英文使用含义一致的简短名称。
- 各段宽度按实际章节时长成比例，不使用等宽假进度。
- 章节文字固定为 `20px`、`#111413`，居中嵌入分段条并始终位于进度色块上层，字号必须明显小于正文内容文字；当前章节加粗，其余章节保持常规粗体，文字颜色不随播放状态变化。
- 已播放区域固定使用浅蓝 `#A8D8F0`，未播放区域固定使用 `#DDE0DA`；两种状态都占满 `52px` 分段条高度，章节文字始终使用 `#111413` 并位于色块上层。
- 章节之间固定保留 `4px` 间隔并露出 `#ECECEA` 底色，保证分割清楚；不得通过加深文字、增加彩色描边或恢复蓝色进度表达分割。
- 章节切换只改变标签状态和进度填充，不做弹跳、放大或遮挡内容的动画。

### HyperFrames 实现

- 进度条是固定 HTML 叠加层，由 HyperFrames/GSAP 根据章节时间轴驱动。
- 进度轨道从画布最左边延伸到最右边，容器使用 `bottom: 0`、`height: 52px` 和 `gap: 4px`；每个轨道本身同样为 `52px` 高，不使用圆角。

## 横屏构图与动效

- 固定 `#ECECEA` 中性纸灰底和极浅同色系网格，使用 `#111413` 粗线条，蓝色、黄色和红色只作少量语义高亮。

- 每个画面只表达一个主判断，并保留一个明确视觉重心。优先使用一个主动角色和一个核心物体，但不得把全片固定成“左侧 Agent + 右侧道具 + 底部文字”；人物、Tiny Agent、一个或两个道具、大号结论文字可以根据内容独立或组合出现。
- 人物、Tiny Agent 和道具都必须获得符合语义的出场机会。允许只有人物居中、只有 Tiny Agent 居中、只有两个道具比较，或把人物/Agent/道具缩到一侧并让巨大粗体结论占据主要空间；同一构图最多连续出现两次。
- 顶部只保留 `Tiny Agent` 和当前主题标题，不显示来源、章节编号、解释性注释或黄色标题下划线。
- 标题使用独立标题带。角色、道具、关系线和比较面板的可见区域必须从标题实际底边以下至少 `24px` 开始；视觉重心校正只能整体移动标题带和内容带，不得让内容重新进入标题带。
- 主画面以工程师和 Tiny Agent 为主体，角色必须参与任务，不得长期作为角落里的静态装饰。
- 字幕位于下方独立区域，不显示“核心”“字幕”等标签。
- 底部章节进度条与字幕分层，不得重叠。
- 除标题、字幕和章节标签外，画面文字尽量减少；原文来源只保留在 `source.md`、`content-map.json` 和发布元数据的结构化 `source` 字段中，不进入公开视频简介。
- 全片目标约 `35-45` 个稳定视觉状态。每个正文状态承载 `2-4` 句连续旁白，通常持续 `10-20 秒`，稳定状态中位时长不低于 `12 秒`；姿势、道具和焦点只在语义变化时切换。超过 `20 秒` 的状态使用渐进揭示或局部焦点变化，但不为制造变化而替换整组道具。
- 每个实质内容章节至少使用 `2` 种构图；时长超过 `60 秒` 的实质内容章节至少使用 `3` 种。同一构图最多连续出现 `2` 次。
- 同一逻辑容器必须先按容器内的元素数量排版，不能直接套用全场元素坐标：`1` 个图标居中；`2` 个图标等尺寸、同一水平线并围绕容器中心对称；无方向语义的 `3` 个图标以容器中心为重心形成等边三角形；`4` 个图标使用规则 `2×2` 网格。链路、分支和循环分别使用横向链、树形和环形几何，不套用数量模板。
- 对角线散点只用于明确表达方向、递进、冲突或因果关系，并且必须有连线或动作承接；没有这类语义时，不得用左上加右下的方式填充空白。
- 正文构图统一经过“逻辑分组 → 语义几何 → 安全区约束 → 可见像素视觉重心校正”四步。主体安全区为 `x=56-1864`、`y=120-820`；角色、道具、标题和比较面板作为一个整体平移，不单独补偿某个元素。
- 正文视觉重心目标为 `(960, 470)`。编译期按透明素材的可见边界和标题字重估算视觉质量，水平偏移不得超过 `160 px`、垂直偏移不得超过 `100 px`；渲染抽帧按实际像素复核，水平偏移不得超过 `150 px`、垂直偏移不得超过 `100 px`。
- 角色位置和大小随构图调度：流程图可退到两侧参与交付，重点页放大 Tiny Agent，主结论页可让角色和结论形成主次关系。可镜像的构图应交替视觉轴线，角色不能全片固定在同一坐标。
- 每 `4-7 秒` 至少出现一次与旁白语义一致的视觉节拍，包括主体聚焦、关系线变化、角色姿势变化或道具状态变化；不得为了满足频率逐句换页。
- 每条视频至少包含 `7` 种实际可见动作类型和 `20` 个动作节点。动作类型可包括画外飞入、短距离推动、放大聚焦、按压回弹、跳动、左右轻摆、旋转进入、浮动和逐项高亮；动作必须绑定旁白语义，并在 `animation-plan.json` 中可追溯。
- 重点内容只对当前角色、道具或关系执行 `1.03-1.10` 倍的短暂聚焦，或直接切换到大标题/大主体构图；不得缩放整个场景制造强调。
- 不做持续镜头推进、频繁缩放和逐句翻页。视觉变化由构图切换、角色调度、重点聚焦和道具状态共同完成。
- 角色姿势使用短位移或短交叉淡化；道具优先留场，通过描边、连接关系和局部缩放改变焦点。
- 流程连接线必须从一个图标的可见边界连接到另一个图标的可见边界，不得穿过透明区域连接中心点；无明确语义时不额外添加焦点圆环。
- 复杂流程先尝试转化为角色行动。明确的比较、步骤、分类或判断结构可以使用简洁的对比、流程和网格构图，但不得退化成满屏文字课件。

### 固定素材与临时生成图

- 每条视频以活跃资产包中的人物、Tiny Agent 和道具为主，同时让临时图片生成的新场景素材占全部非结束页视觉状态约 `15%-20%`，不得低于 `15%`。临时素材必须符合纸灰白板风格和当期语义，不能只用固定素材重复拼装。
- 固定内部道具和临时生成图都直接融入 `#ECECEA` 背景，不增加外层卡片、黑色边框、圆角容器或投影。只有开头用于承载权威背书的完整信息卡可以保留结构性描边；普通正文道具不得借此例外恢复卡片框。
- 临时生成图与黄色高亮文本同时出现时必须分处画面两侧，并为文字保留独立空间。图片在左则文字在右，图片在右则文字在左；同一视频按场景交替左右轴线，不得全部固定在一侧。
- 黄色高亮文本容器必须根据文本实际宽高自适应。优先保持完整短句单行；需要换行时按语义接近对半拆分，不留下单字、单个短词或标点孤行。中文固定宽度高亮框单行默认不超过 `11` 个非空白字符；英文按完整单词断行，并以实际 DOM 包围盒为最终准出依据。
- 所有黄色高亮文本必须执行真实浏览器 DOM 包围盒检查：每一行的 `left/right/top/bottom` 都必须位于容器边界内。任何文字越过边框、被裁切或与图片重叠都直接阻塞渲染和发布，不得只依赖字符数估算。

### 时间轴优先编排

- 固定顺序为：分析来源并完成脚本 → 生成最终 TTS 与 VTT → 建立段落和字幕时间刻度 → 将场景、角色、道具和重点动画放入对应时间段 → 编译视频。
- 双语版本分别执行完整时间轴流程。英文版和中文版不得共享 TTS、VTT、`timing-map.json` 或 `animation-plan.json`；只能共享内容地图、章节语义、角色任务和构图意图。
- 英文或中文脚本为适应自然表达允许调整句序和局部例子，但中心主旨、内容优先级、章节承诺、可复用产物和事实边界必须一致。
- `scene-plan.json` 只负责声明脚本段落与视觉语义的对应关系，不自行估算时长，并必须把首句标记为 `voice-synced-kinetic-question`、把前言最后的关注收藏场景标记为 `intro-follow-save`；`timing-map.json` 必须记录首句字级/词内字母级触发点、问题卡结束点、前言末尾关注收藏场景、段落、道具、小结和片尾关注引导的真实时间。
- 场景边界必须落在 VTT 句子或脚本段落边界。道具优先在对应关键词所在字幕开始时出现；没有直接关键词时，才使用该场景真实字幕提示点的语义分位。
- 每条视频必须生成 `animation-plan.json`。每个道具只能归属一个语义节拍，节拍必须绑定最终旁白中真实存在的短语；编译器从最终字幕时间轴计算出现时刻，禁止根据道具序号、字幕数量或场景时长均分。
- 同一短语共同表达一个概念的道具作为一组同时出现；只有旁白明确讲述先后步骤时才逐项出现。列表在同一句中快速读出、剩余展示时间不足或逐项出现没有新增含义时，必须合并为一个节拍。
- 每个语义节拍出现后至少保留 `1 秒`可读时间。触发短语过晚时，应提前合并到上一个完整语义组或调整场景边界，不得把动画强行挤在转场前。
- 不允许先按平均时长铺画面，再让旁白迁就画面。脚本或旁白变化后，必须重新生成 VTT、`animation-plan.json` 和全部视觉时间锚点。

### 构图类型

| 类型         | 适用内容                 | 画面规则                                                                          |
| ------------ | ------------------------ | --------------------------------------------------------------------------------- |
| `hero`       | 章节主结论、反常识判断   | 结论放大，角色与一个主道具形成强主次                                              |
| `process`    | 顺序、循环、工具调用     | 横向或环形连接，角色位于流程起点和结果端                                          |
| `comparison` | 方案差异、适用边界       | 左右双区分别计算布局；每区 `1` 个证据居中、`2` 个证据水平并列，不跨区使用对角坐标 |
| `grid`       | 分类、问题清单、并列能力 | `2-5` 个单元形成稳定网格，角色负责指向或确认                                      |
| `focus`      | 风险、结论、记忆点       | 主体明显放大，其他信息降级为辅助证据                                              |

### 转场规则

- 同构图连续场景使用 `0.45-0.50 秒`的短距离推移与遮罩揭示；不同构图使用 `0.50-0.55 秒`的遮罩揭示；章节切换使用 `0.70-0.80 秒`的纵向覆盖揭示。
- 转场以一个主运动为准，禁止旧画面和新画面分别渐出渐入。新画面覆盖完成后必须立即结束旧场景生命周期。
- 章节开场向正文使用遮罩揭示，章节正文向总结使用侧向揭示；不使用全屏黑场或白场闪切。
- 字幕先在 `0.1 秒`内退出，下一条再用约 `0.18 秒`进入，两个字幕文本不得同时可读。
- 转场前后主体位置需要有连续关系；只有章节边界允许明显改变视觉轴线。
- 正文转场不得反复让整张画面滑出画布；强调动画不得带动字幕、进度条或整个场景缩放。
- 新场景中的角色、道具和标题必须从遮罩揭示起点开始入场，禁止先完整出现再执行入场动画。
- 新场景揭示前 `0.02 秒`必须先写入角色、标题、比较面板和章节卡的隐藏起始状态，再执行单向进入动画；不得使用延迟启动且 `immediateRender:false` 的 `fromTo` 让元素先以最终状态暴露。
- 角色入场只使用 `8-10px`短上浮和透明度变化，持续约 `0.56 秒`；场景遮罩已经承担主运动时，不再叠加角色缩放。人物与 Tiny Agent 可错开不超过 `0.04 秒`，但都必须在画面揭示前完成状态预置。

| 维度       |  权重 | 判断                                           |
| ---------- | ----: | ---------------------------------------------- |
| 点击吸引力 | `35%` | 是否提出目标观众正在经历的问题、损失或结果差异 |
| 搜索相关性 | `30%` | 主搜索词是否自然出现，是否符合真实搜索表达     |
| 理解成本   | `20%` | 新观众能否在一次扫读中理解视频价值             |
| 内容兑现度 | `15%` | 开头和正文是否完整兑现标题承诺                 |

### 关键词与话题

- 每期只选择 `1` 个主关键词和 `3-5` 个次关键词。次关键词分别覆盖：上位概念、具体问题、预期结果、来源实体或常见英文同义词；不是每类都必须使用。
- 主关键词自然出现在发布标题、简介前两行和对应章节标题中；次关键词分布在简介正文、章节和实际旁白中，不额外生成重复关键词列表。
- 关键词必须来自当期内容，不得为了流量固定附加 `ChatGPT`、`Claude`、`Gemini`、`OpenAI` 等无关产品名。
- YouTube `tags` 字段只填写 `5-10` 个拼写变体、中英文对应词或容易写错的专有名词，不把 tags 当作主要流量来源。
- 发布后每周根据 YouTube Analytics 的真实搜索词、点击率和观看时长更新词库；不得只根据模型猜测长期固定关键词。

- “What you'll learn”或“你会学到”固定为 `3` 点，并与对应语言的开头概览、正文重点和结尾总结一致。
- 英文 YouTube 简介的章节名称来自英文最终视频章节，时间戳来自英文 `timing-map.json`，不得按脚本字数估算。中文 `timing-map.json` 仍用于视频与动画准出，但章节时间不得写入中文平台简介。
- `publish-metadata.<locale>.json` 可以保留结构化 `source` 字段供归档和核验，但 `description` 必须通过无 `来源：`、无机构/原文标题拼接、无 `http://`/`https://` 的检查。

## 准出检查

### 内容与脚本

- 已分析完整来源，不是逐段缩写或按文字量分配时长。
- 中心主旨、P0 和 P1 内容全部进入视频；删除项有明确原因。
- 双语脚本已完成内容编辑检查：权威来源与必要证据完整保留；删除项均确认没有增加事实、证据、判断、边界、步骤、必要例证或理解帮助，不存在为命中时长而机械删减高价值内容。
- 首帧是第一句问题的全屏逐字动效且无淡入，不展示正式封面；问题文字与首条 VTT 一致并在 `5 秒`内念完，中文字符和英文词内字母比对应语音触发点提前 `0.10-0.15 秒`开始出现。
- 首句问题卡使用不低于 `800` 的超粗体和安全区内最大字号，按语义合理换为 `2-3` 行；完整问题可见包围盒建议不低于画布宽度 `82%`、高度 `52%`，底部无无效大块留白。Tiny Agent 主动避让文字，DOM 包围盒证明无裁切、无溢出、无单字或单词孤行。
- 问句念完后完整问题保持可读 `1.2-1.6 秒`再硬切正文，正文第一帧同时显示章节进度条；`retention-opening-report.json` 记录视觉提前量、完整问题停留时间、硬切时间和问题包围盒。
- 首句问题卡至前 `30 秒`内容门槛均不包含关注、收藏或保存引导；权威来源不晚于 `12 秒`，具体损失和收益承诺不晚于 `20 秒`，可复用产物预览和实质解释不晚于 `30 秒`。随后在前言最后一个 `intro-follow-save` 场景完整呈现对应语言固定关注收藏画面和文案，再进入第一个实质章节。
- 总结只复述中间讲过的关键点，并提供可记忆的判断、步骤或公式。
- 中英文关注引导分别严格使用对应语言的固定文案；英文旁白逐字念出 `Follow Tiny Agent. Tiny Agent helps you get better at using AI.`，中文旁白逐字念出 `关注 Tiny Agent，成为更擅长使用 AI 的人！`，都不根据当期主题改写，不承诺下一期具体内容。
- 所有来源结论可回溯；补充示例没有被描述成来源事实。
- 目标时长符合深度内容档位，最终 TTS 落在 `9-12 分钟`内；双语声音均为规定音色和固定 `+10%` 语速。前 `30 秒`普通句间停顿不超过 `0.2 秒`，正文使用约 `0.25/0.5/0.8 秒`的固定处理停顿。
- 全片使用一条普通人熟悉的日常故事贯穿不超过三个实质模块；抽象知识约 `20 秒`内回到案例，陌生术语首次出现前有一句通俗解释，结尾用同一可复用工具重新解决开头故事。
- `qa/cognitive-load-report.json` 和 `qa/recap-report.json` 均为 `pass: true`；稳定视觉状态约 `35-45` 个、中位时长不低于 `12 秒`，没有正式章节三点口播小结。
- 英文最终发布标题、三个英文候选、英文视频内标题和英文封面标题均自然包含 `AI Agent` / `AI Agents`；中文本地标题、中文视频内标题和三张中文封面标题均自然包含 `AI Agent` / `智能体`。所有标题脱离频道上下文后仍能识别主题对象和本期内容，不存在只剩情绪或判断、看不出 AI Agent 主题的孤立口号。
- 发布标题的核心问题与开头钩子一致，标题承诺在正文中得到完整兑现，没有原文标题直译、关键词堆砌或误导表达。
- 简介前两行包含当期主关键词和明确收益；英文 YouTube 简介包含三点收益、真实章节、固定关注句和 `4-6` 个相关 hashtag；中文本地简介包含三点收益、固定关注句和相关 hashtag。两种语言的 `description` 均无来源段、来源机构/标题拼接及 `http://`、`https://` 外站链接；中文元数据不存在小红书专用字段或平台表单完成状态。
- 英文和中文版本的脚本、旁白、字幕、章节时间和动画均独立对齐；不存在跨语言复用时间戳。
- `publish-metadata.en-US.json` 与英文最终视频、章节时间、来源和英文 `summary.json` 一致；`publish-metadata.zh-CN.json` 与中文本地视频及中文 `summary.json` 一致。

### 角色、画面与进度条

- 工程师和 Tiny Agent 外形稳定，姿势交换使用统一锚点，没有位置跳动。
- 角色承担讲解动作，主画面不是静态角色加 PPT 内容。
- 每个主要道具至少承载一组连续旁白，不逐句更换道具。
- 每个实质内容章节存在章节开场，并以一句因果收束或待解决问题自然进入下一段；开场承诺、正文重点和转场问题属于同一故事链。
- 中文和英文正文均不存在固定三点章节口播小结，不朗读 `本章小节` 或 `Chapter recap`；完整知识回收只在全片总结中出现。
- 每个实质内容章节满足构图多样性要求，没有超过 `2` 个连续同构图场景。
- 重点内容存在明确的尺度或构图调度，不依赖常驻文字重复强调。
- 画面没有来源标签、解释性杂项、黄色标题下划线或字幕标签；章节编号只出现在章节开场，不进入底部进度条。
- 章节进度条包含 `前言`、全部实质内容章节和 `总结`，标签按最终顺序连续显示 `1. 名称`、`2. 名称`、`3. 名称` 等数字前缀。
- 进度条不出现“钩子”“概览”“关注引导”“片头”“片尾”。
- 各章节宽度与实际时长成比例；导航为底部全宽 `52px` 满高分段，章节间隔为 `4px`，文字为 `20px` 黑色且位于色块上层。
- 已播放区域为浅蓝 `#A8D8F0`，未播放区域为 `#DDE0DA`；两个状态清楚可辨，文字保持黑色并位于填充层上方。
- 进度条、字幕、角色、标题和关键道具没有重叠或越界。
- 临时生成图场景占非结束页视觉状态 `15%-20%` 且不低于 `15%`；整条视频至少包含 `7` 种动作类型和 `20` 个动作节点。
- 普通内部道具与临时生成图没有外层卡片、边框、圆角或阴影；临时图与黄色高亮文本始终分处两侧，并在全片交替左右轴线。
- 全部黄色高亮文本通过真实浏览器 DOM 包围盒检查，失败数为 `0`；不存在文字越过边框、裁切、单字孤行、图片与文本重叠或不自然断行。
- 所有正文场景通过标题带隔离校验，标题底边与任何角色、道具、关系线或比较面板之间至少保留 `24px`。
- 每个逻辑容器中的双图标保持等尺寸、同一水平线、间距一致并围绕容器中心对称；编译期布局校验通过。
- 无方向语义的三图标组形成等边三角形；全部正文场景通过安全区、视觉重心和实际像素平衡校验，运行 `pnpm run check:balance` 后失败场景数为 `0`。
- `animation-plan.json` 覆盖全部正文道具；运行 `pnpm run check:semantics` 后语义兜底数、重复绑定数、未绑定数和标题带失败场景数均为 `0`。
- 运行 `pnpm run check:transitions` 后，全部角色入场均在内容揭示前完成状态预置，延迟角色 `fromTo` 数和失败角色数均为 `0`。
- 所有流程连接线都止于图标可见边界，没有穿过图标中心、文字标签或角色。
- 进度轨道横向覆盖完整画布，首段从 `x=0` 开始，末段在 `x=1920` 结束。
- 章节开场和自然转场期间常驻页眉隐藏，字幕与进度条仍可读；独立关注结束页隐藏常驻页眉、字幕框和章节进度条。
- 场景切换没有双重渐隐、短暂黑场、旧画面残留、角色先出现后回跳或字幕双行叠加。

### 音视频与 HyperFrames

- 成片为 `1920x1080`、`30fps`，包含 H.264 视频和 AAC 音频。
- 音频清楚，无削波、爆音、明显机械断句或静音尾巴；目标综合响度为 `-16 至 -18 LUFS`。
- 字幕与旁白同步，单屏不超过两行，长句按语义换行。
- `timing-map.json` 记录固定 CTA 的 VTT 起点；结束页首帧与该起点误差不超过 `1/30 秒`。英文结束页完整显示 `Follow Tiny Agent` 和英文收益句，中文结束页完整显示 `关注 Tiny Agent` 和 `成为更擅长使用 AI 的人！`；两条音轨都必须念出各自的完整固定文案。
- `timing-map.json` 证明场景、自然章节转场、道具节拍和最终总结来自最终 VTT，而不是字数比例。
- `pnpm dlx hyperframes check . --snapshots --frame-check=severity=error` 通过。
- 在首帧全屏问题卡、首句结束后的下一帧、前言末尾 `intro-follow-save` 场景、该场景结束后的下一帧、每个章节中点、章节切换点和结尾抽帧；确认问题文本与 VTT 一致、开头未出现封面、关注收藏场景位于前言最后且文案完整，并确认无黑帧、空白主画面、角色跳动、文字溢出或进度错误。
- 最终 MP4 存在且大小合理，`ffprobe` 的分辨率、帧率、时长和音视频流符合规则。
- 任一准出项失败时停止发布，修复源脚本、资产、时间轴或布局后重新生成。
