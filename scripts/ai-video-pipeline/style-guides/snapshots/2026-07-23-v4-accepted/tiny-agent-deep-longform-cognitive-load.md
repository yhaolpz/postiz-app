# Tiny Agent 深度长视频叙事与认知负担基线

## 适用范围与优先级

本规则用于 `AI Agent Deep Dives / AI Agent 深度方法` 横屏长视频，英文版和中文版同时生效。两种语言共享中心主旨、知识优先级、故事因果链、可复用工具和视觉语义，但必须分别完成自然脚本、TTS、VTT、时间轴、场景编排和动画。

本规则优先于历史工程中的快语速、`5-8 分钟`时限、逐句换场和每章三点口播小结。标题身份、纸灰视觉、Tiny Agent 资产、双语封面、前言末尾关注收藏、独立结束页和英文发布边界继续执行 `tiny-agent-longform.md`。

## 内容组织

### 保留知识，降低接收成本

- 保留全部具有独立价值的 P0、P1 和 P2：事实、证据、机制、边界、判断、步骤、必要案例和迁移方法都不得为了压时删除。
- 只删除不增加新信息的重复解释、重复小结、客套转场和相似案例。案例用于承载知识，不得取代知识。
- 深度视频优先组织为三个连续心智模块：`机制 → 诊断 → 行动`。章节名可按主题自然命名，但因果顺序必须清楚。
- 一个视觉状态只要求观众完成一个心智动作，最多引入两个新抽象概念。抽象讲解连续超过约 `30 秒`时，必须回到案例、画面演示、判断题或可执行动作。

### 一条日常故事贯穿全片

- 选择不需要行业背景、所有普通观众都能立即理解的日常情境，例如带伞、订餐、出行、购物、预约或家庭安排。不得把订单号、内部流程、企业系统字段或陌生业务术语作为主案例入口。
- 全片只保留一条核心故事线。故事先产生一个观众在意的结果，再逐步揭示原因、判断方法和行动选择；不得把多个互不相关的小例子连续堆叠。
- 每个抽象规则在约 `20 秒`内回到同一故事，展示它如何改变判断。案例应替代抽象复述，而不是在知识讲完后额外加一段。
- 模块之间使用前一段尚未解决的问题自然承接，例如“它为什么会猜”“它现在有没有条件知道”“证据是否足以支持行动”。不得用割裂的目录播报重新起头。
- 不设置每章固定三点口播小结，不朗读“本章小节”或 `Chapter recap`。模块结尾用一句因果收束或一个待解决问题进入下一段；完整知识回收只放在全片总结。

### 术语与能力获得感

- `SimpleQA`、校准、可答性等陌生术语首次出现时，先用一句普通人能听懂的话说明“它是什么、用来判断什么”，再给数据或正式定义。
- 例子之后必须明确抽取知识：模型为什么会这样、证据说明了什么、边界在哪里、观众下一步怎么做。不得只讲完故事就进入下一个故事。
- 每期必须交付一个可以记住并立即使用的能力模型，例如“三道门”“四种证据状态”或“一张检查表”。工具应统一吸收正文知识，不得与正文平行再造一组概念。
- 前言明确说明学完后观众能判断或完成什么；结尾用同一工具重新解决开头故事，让观众感受到自己已经获得解决问题的能力。

## 固定语音与处理节奏

- 中文固定使用 `zh-CN-YunxiaNeural`、`rate +10%`；英文固定使用 `en-US-AnaNeural`、`rate +10%`。不得按选题、章节或时长动态提速。
- 正文普通句界停顿约 `0.25 秒`，知识单元边界约 `0.5 秒`，章节边界约 `0.8 秒`。开头前 `30 秒`仍执行留存优先规则，普通句间停顿不超过 `0.2 秒`。
- 深度视频成片目标为 `9-12 分钟`。不足 `9 分钟`不靠重复解释补时；超过 `12 分钟`先删除无信息增量的重复内容，仍超出则拆成两个独立主题。
- 中英文分别生成最终 TTS/VTT 后再计算实际时长和全部节拍，禁止用字数估算时长或跨语言复用时间戳。

## 首句全屏问题动效

- `0 秒`直接进入纸灰手绘逐字问题场景。问题是第一视觉主体；正式封面、字幕框、常驻页眉和章节进度条全部隐藏。
- 问题优先排为 `2` 行；内容需要时中文和英文均可使用 `3` 行。按语义短语断行，不拆英文单词，不留下孤立冠词、介词、单字或标点。
- 字号使用安全区内能容纳的最大值，不为 Tiny Agent 或装饰预留大片空间。完整问题的可见包围盒应尽量覆盖主内容安全区，建议宽度不低于画布 `82%`、高度不低于画布 `52%`；底部不得保留没有语义用途的大块空白。
- Tiny Agent 是辅助主体，应缩小、右移或调整入场路径避让文字；不得为了保留角色尺寸而缩小问题字体。
- 中文每个汉字和标点、英文每个单词内字母均比对应语音触发点提前 `0.10-0.15 秒`开始出现，让观众先看到再听到，但不得提前展示后续完整单词或整句。
- 第一问仍须在 `5 秒`内念完。最后一个字形和问号稳定后，完整问题保持可读 `1.2-1.6 秒`再硬切正文；停留期间不得插入新文案、黑帧或关注收藏引导。
- 两种语言按各自最终 TTS/VTT 独立计算视觉提前量和完整问题停留时间。`retention-opening-report.json` 必须记录并验证 `maximumVisualLeadSeconds`、`fullQuestionHoldSeconds`、问题包围盒、裁切、遮挡和硬切时间。

## 视觉信息节拍

- 全片目标约 `35-45` 个稳定视觉状态，通常每个状态持续 `10-20 秒`，稳定状态中位时长不低于 `12 秒`。不为每句话单独换场。
- 一个状态承载 `2-4` 句相关旁白，通过逐项出现、短推、局部聚焦、颜色切换或角色动作渐进揭示，不一次铺满全部信息。
- 每个状态只有一个明确视觉重心，同时可读文字层不超过两层，字幕计入其中。正文继续执行标题、角色、道具和底部进度条的安全区规则。
- 同一故事中的人物、Tiny Agent 和核心道具保持空间与状态连续。只有语义任务改变时才切换构图，避免观众反复重新寻找视觉入口。
- 画面必须交替使用人物、Tiny Agent、道具、结论文字、生成图和流程关系；同一构图最多连续两次，但不得为追求变化而牺牲故事连续性。

## 双语内容一致性

- 中英文共享同一条日常故事、同一因果链、同一知识覆盖、同一事实边界和同一最终工具。
- 两版都必须自然讲述，不机械逐句翻译。英文可调整句序和比喻，中文可调整语气与断句，但不得改变案例结果、证据含义或行动门槛。
- 两版分别进行术语科普、语速验证、场景时长检查和开头 DOM 检查。英文不得因为单词更长而缩小为普通字幕式问题卡；应先优化问句和断行，再使用最大安全字号。

## 准出要求

双语项目除既有 QA 外，必须生成 `qa/cognitive-load-report.json` 和 `qa/recap-report.json`，并满足：

- 深度视频时长在 `9-12 分钟`。
- TTS 声音和语速严格为对应语言固定值 `+10%`。
- 实质模块不超过三个；前言和总结不计入。
- 一条普通人熟悉的核心故事贯穿全部实质模块，抽象知识回到案例的间隔不超过约 `20 秒`。
- 陌生技术术语首次出现前存在一句通俗说明。
- 约 `35-45` 个稳定视觉状态，中位时长不低于 `12 秒`，大多数状态落在 `8-24 秒`舒适区间。
- 不存在正式章节三点口播小结；模块间使用自然因果过渡，最终总结完整回收关键知识和可复用工具。
- 首句文字视觉提前量为 `0.10-0.15 秒`，完整问题停留为 `1.2-1.6 秒`，文字尽量铺满安全区且无裁切、溢出或角色遮挡。
- 成片抽查首句逐字出现、完整问题停留、正文第一帧、故事关键转折、每个模块中点、模块衔接、最终工具回收和独立结束页。

任一硬门槛失败时，不得发布英文版，也不得把中文版标记为完成。
