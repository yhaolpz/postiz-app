# Generation prompts and provenance

## Character provider

- Mode: built-in `image_gen`.
- Human references: v1 human contact sheet and `sprites/human/idle.png`.
- Agent references: v1 Agent contact sheet and `sprites/agent/idle.png`.
- Output: one generated image per new pose on a flat `#00ff00` chroma-key background, then local background removal and anchor normalization.

The exact action and direction for every call come from `catalog.mjs`. The shared character prompt requires one isolated full-body subject, unchanged identity, thick black marker outlines, white fills, sparse blue/red accents, no external props, and no text, labels, scenery, floor, shadow, or watermark.

Direction language is viewer-relative:

- Left: gesture, gaze, and body clearly target the left side of the image.
- Right: gesture, gaze, and body clearly target the right side of the image.
- Front: face squarely forward without favoring either side.

## Props provider

The 60 new props are deterministic SVG-derived filled whiteboard objects generated by `tools/build-props.mjs` from the repository's installed `lucide-react` geometry, then rasterized to 320x320 sRGB PNG with `rsvg-convert`. Existing v1 props remain unchanged. New props match the v1 visual language: rounded black contours, white paper bodies, semantic blue/yellow/red fills, subtle light-to-dark fill depth, transparent backgrounds, and no detached decorative dots or underlines.

This code-native path is intentional: the prop library remains reproducible while each prop has an explicit semantic fill map. A generic outline-only icon treatment is not an acceptable fallback.
